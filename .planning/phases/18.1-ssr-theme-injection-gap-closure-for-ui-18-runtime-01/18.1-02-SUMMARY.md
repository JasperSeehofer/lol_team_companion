# Phase 18.1 Plan 02: e2e Re-baseline & Threshold Calibration Summary

Updated the e2e test infrastructure to be SSR-aware now that Wave 1 made the `lol_companion_theme` cookie authoritative: rewrote the `setRegion` Playwright helper from "click toggle + wait for WASM Effect" to "addCookies + reload", re-captured all 24 scoped visual-regression baselines under the post-18.1 runtime (revealing actual structural Pandemonium branches for the first time in test artifacts), and calibrated `REGION_DIFF_THRESHOLD` from 0.005 (pre-18.1 token-only floor) to 0.014 (post-18.1 viewport-measured floor with 30% headroom).

## setRegion: Old vs New

**OLD body** (pre-18.1 — only flipped client tokens, SSR still rendered demacia):

```typescript
const themeAttr = await page.getAttribute("html", "data-theme");
if (themeAttr === region) return;
const btnText = region === "pandemonium" ? "Pandemonium" : "Demacia";
await page.click(`button:has-text("${btnText}")`);
await page.waitForTimeout(700); // wasm-patterns rule 56
const newTheme = await page.getAttribute("html", "data-theme");
if (newTheme !== region) throw new Error(...);
```

**NEW body** (post-18.1 — SSR honours the cookie on next render):

```typescript
await page.context().addCookies([{
  name: "lol_companion_theme",
  value: region,
  domain: "127.0.0.1",
  path: "/",
  sameSite: "Lax",
}]);
const currentUrl = page.url();
if (currentUrl && currentUrl !== "about:blank") {
  await page.reload({ waitUntil: "networkidle" });
  await page.waitForTimeout(500);
  const themeAttr = await page.getAttribute("html", "data-theme");
  if (themeAttr !== region) throw new Error(...);
}
```

Backwards compatibility: every existing call site (in `region-diff.spec.ts` and `visual-regression.spec.ts`) continues to work unchanged. Callers that invoke `setRegion` BEFORE `goto()` get the cookie attached to the browser context first; callers that invoke AFTER `goto()` get a transparent reload.

## Deviations from Plan

### [Rule 1 - Bug] Playwright `addCookies` rejects `url + path` combo

**Found during:** Task 2 — first `--update-snapshots` run.

**Issue:** All 24 scoped tests failed with `browserContext.addCookies: Cookie should have either url or path`. Playwright requires exactly ONE of `url` (which derives domain+path) OR explicit `domain + path` — not both. The plan's Task 1 spec inlined `url: "http://127.0.0.1:3020"` AND `path: "/"`, hitting this constraint.

**Fix:** Switched to `domain: "127.0.0.1"` + `path: "/"` (the explicit pair). Bonus: this form also works when `setRegion` is called BEFORE any `page.goto()` (the `about:blank` case the plan flagged as an edge case), because the cookie is scoped explicitly to the dev-server domain regardless of the page's current URL.

**Files modified:** `e2e/tests/fixtures.ts`. Folded into the Task 2 commit (`c453e06`) since the broken cookie format only surfaced when the test suite tried to actually use the helper.

**Commit:** `c453e06`

### [Rule 4 - Spec band mismatch] REGION_DIFF_THRESHOLD landed at 0.014, outside plan's [0.03, 0.10] estimate

**Found during:** Task 3 — ratio measurement run.

**Issue:** The plan's `<verification>` block asserts the threshold should land in `[0.03, 0.10]`. The empirically measured MIN_OBSERVED was 0.0206 (`/draft war-table`), yielding `0.0206 × 0.7 = 0.0144 ≈ 0.014` — below the estimated floor.

**Why this is Rule-resolved, not a checkpoint:** CONTEXT.md D-08 explicitly states "Final value is empirical, not pre-committed" and the plan's Task 3 action 4 says "Set `REGION_DIFF_THRESHOLD` to `MIN_OBSERVED * 0.7`". The plan-spec is internally consistent on the principle (empirical wins, formula authoritative) and inconsistent on the estimate (the [0.03, 0.10] band assumed all routes would exhibit 5–15% divergence, which empirically did not hold for the /draft viewport cluster). Honoured the formula per the spec_lock; documented the estimate as superseded.

**Why the /draft cluster is lower than expected:** /draft is screenshotted viewport-only (1280×720) per RESEARCH.md Pitfall 6 (regions have different page heights). The /draft Pandemonium structural branches (RiotTape "DRAFT_LEDGER · v0.1", Glitch eyebrow text, zine cards) render BELOW the 720px viewport fold. The upper viewport (player slot strip + champion-picker carousel header) is more layout-aligned across regions. Confirmed by the structural divergence spot-check above: fullPage /draft ledger Demacia is 2584px vs Pandemonium 2282px, ~95%+ differing pixels — the divergence exists, just below the diff window.

**This is NOT a Phase-18 source bug** — the source code does branch on `is_pandemonium`; the viewport-vs-fullPage screenshot mode dichotomy is the only reason the ratio is "small" for /draft.

**Files modified:** `e2e/tests/region-diff.spec.ts` comment block updated to document this rationale.

**Commit:** `9858d0b`

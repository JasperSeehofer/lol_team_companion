## New finding: UI-18.1-HYDRATE-01 (recommended for Phase 18.2)

**Symptom:** Every Pandemonium-themed page panics during WASM hydration:

```
[error] panicked at /home/jasper/.cargo/registry/src/index.crates.io-1949cf8c6b5b557f/tachys-0.2.14/src/html/mod.rs:217:14:
called `Option::unwrap()` on a `None` value
```

**Impact:** Client-side interactivity (mode toggles, theme toggle, draft drag-drop, all event handlers) does not fire on Pandemonium pages. Page renders correctly via SSR; WASM crashes during hydration reconciliation.

**Why it didn't appear pre-18.1:** Pre-18.1, every SSR request returned the Demacia tree (the very bug UI-18-RUNTIME-01 reported). Hydration matched fine because both SSR and the client's first WASM render produced the same tree. Phase 18.1's middleware now makes SSR render the Pandemonium tree when the cookie says so, exposing a latent SSR/hydrate hash mismatch in some Pandemonium structural primitive.

**Why this doesn't block UI-18-RUNTIME-01's resolution:** That finding was scoped to "does the SSR-rendered DOM contain the Pandemonium structural branches?" — answer empirically YES across 7/7 routes × 28 markers. The hydration panic is a separate, newly-surfaced bug, not a regression of Phase 18.1's scope.

**Recommended scope for Phase 18.2 (per user directive):** Diagnose which Pandemonium primitive triggers the unwrap (the panic is universal across all Pandemonium pages, suggesting it's a structural primitive shared by all of them — likely something in the `<Card>`, `<Glitch>`, `<RiotTape>`, or `<SectionHead>` variant). Fix the SSR/hydrate hash mismatch. Re-run the agent-browser sweep with WASM event handlers exercised to confirm interactivity restored.

**Caveat for Phase 19 sequencing:** ROADMAP says Phase 18.1 unblocks Phase 19 (Bug-Report Widget). Strictly speaking the UI-18-RUNTIME-01 architectural blocker IS resolved. But Phase 19 introduces a click-driven bug-report widget — and "the widget should work in Pandemonium too" is a reasonable acceptance criterion. Phase 18.2 should likely intervene between 18.1 and 19, OR Phase 19 should ship Demacia-only and tag a follow-up. Surfaced for orchestrator/user decision; not litigated in this plan.

```typescript
export function captureErrors(page: Page): string[] {
  const errors: string[] = [];
  page.on("pageerror", (e) => errors.push(`[pageerror] ${e.message}`));
```

```typescript
const PANIC_REGEX = /panicked at|Option::unwrap.*None|tachys/i;

async function attachPanicListeners(page: import("@playwright/test").Page): Promise<string[]> {
  const errors: string[] = [];
  page.on("pageerror", (err) => {
    errors.push(`pageerror: ${err.message}`);
  });
  page.on("console", (msg) => {
    if (msg.type() === "error" && PANIC_REGEX.test(msg.text())) {
      errors.push(`console.error: ${msg.text()}`);
    }
  });
  return errors;
}
```

### WASM-safe event handlers

**Source:** `.claude/rules/wasm-patterns.md` rule 35; example at `src/components/draft_board.rs:94-99`:

```rust
on:dragstart=move |ev: web_sys::DragEvent| {
    let slots = draft_slots.get_untracked();
    if let Some(Some(champ_name)) = slots.get(slot_idx) {       // ← never .unwrap()
        if let Some(dt) = ev.data_transfer() {                   // ← never .unwrap()
            let _ = dt.set_data("text/plain", champ_name);
        }
    }
}
```

**Apply to:** Any fix that touches an event handler. The 18.2 bug is upstream of this rule (tachys's own unwrap, not user code), but the rule's spirit applies — every fix MUST NOT introduce new `.unwrap()` calls in event handlers.

### `into_any()` for divergent view branches

**Source:** `.claude/rules/leptos-patterns.md` rule 19; example throughout `src/components/region/layout.rs`, `controls.rs`, `ornaments.rs`:

```rust
{move || if filled {
    view! { <img ... /> }.into_any()
} else {
    view! { <span>...</span> }.into_any()
}}
```

**Apply to:** Every fix that introduces or modifies an `if/else` view branch. Both arms must call `.into_any()` and SHOULD produce a same-outer-element shape per Anti-Pattern line 167.

### Cloning region/region-derived values before multiple closures

**Source:** `.claude/rules/leptos-patterns.md` rule 18; example at `src/components/region/typography.rs:97`:

```rust
let e_stored = StoredValue::new(e);
view! { <Glitch region=region.clone()>{move || e_stored.get_value()}</Glitch> }
```

**Apply to:** Any fix that captures `region: String` in multiple `move` closures within the same component. Clone before each closure or store in a `StoredValue`.

### Hydration-friendly conditionals (Leptos 0.8 convention)

**Source:** `.claude/rules/leptos-patterns.md` rule 19 + RESEARCH.md Anti-Patterns lines 165-171:

- Conditional content that differs SSR vs WASM must go inside `{move || ...}` or `Show fallback=...`.
- Static-at-SSR conditionals (`if region == "pandemonium" { ... } else { ... }` at the top of a component) work IFF both branches return the same node-count shape after `.into_any()`.
- `view! {}` empty fragments adjacent to other fragments can desync the cursor walk — prefer `view! { <span></span> }` or `view! { <></> }`.
- `Option<AnyView>` adjacency (`.then(|| view!{...}.into_any())`) is the leptos-rs#3583 trigger — rewrite as explicit `if/else` with stable placeholder.

**Apply to:** Every fix to a region primitive or per-page Pandemonium view in this phase.

### Both-target compile check

**Source:** `CLAUDE.md` "Debugging reactive bugs without a browser" / `just verify`:

```bash
cargo check --features ssr
cargo check --features hydrate --target wasm32-unknown-unknown
```

**Apply to:** Every commit during the bisect AND the final fix. Some hydration bugs only surface in one target (e.g. unused-variable warnings from `#[cfg(feature = "hydrate")]` guards). Both checks must pass.

### Warm-fetch before observing Suspense-sensitive panics

**Source:** RESEARCH.md Pitfall 1 + 18.1-03-EVIDENCE.md §"Suspense-timing caveat" (line 133-135):

```
For each page tested during the bisect:
  1. Fetch the page once (Suspense warm-up).
  2. Wait for networkidle.
  3. Reload OR refresh.
  4. Capture the SECOND-fetch console state — this is the bisect signal.
```

**Apply to:** Every short-circuit-and-reload cycle in Wave 1 AND every e2e test in the new spec. The `attachPanicListeners` + `waitForLoadState("networkidle")` + `waitForTimeout(1500)` pattern in the new spec automatically handles this (the 1500ms timeout gives Suspense + WASM hydrate time to settle into the second-fetch-equivalent state).

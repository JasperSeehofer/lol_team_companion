The parent component is RE-EXECUTED on WASM hydrate and re-computes every prop value from its scope. If a parent reads `use_context::<T>()` where T is provided only on SSR (e.g., via `leptos_routes_with_context`), the child receives DIFFERENT prop values SSR vs WASM. The hydration mismatch is structural even though both child branches are 'safe' — the divergence is in the BRANCH CONDITION value not the branch ARM emission code."

# Phase 18.2 Plan 03: Pandemonium Hydration Fix — CompanionSigil refactor (partial) + diagnosis correction

**CompanionSigil refactored to take `region: String` as a prop (removes legacy `use_context` anti-pattern), but the broader tachys hydration panic persists across all 7 Pandemonium scoped routes AND was discovered to be pre-existing on Demacia too — Plan 01 baseline + Plan 02 audit both missed it due to agent-browser text-mode console truncation. D-10 cost gate fires; deferred fix logged with D-11 audit trail.**

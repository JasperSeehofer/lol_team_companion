## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] RiotTape short-circuit required body replacement, not early return**
- **Found during:** Cycle 4 (RiotTape mutation)
- **Issue:** RiotTape's signature is `-> impl IntoView` and it has no `is_pandemonium` gate (it's Pandemonium-only, called from inside Pandemonium arms). An `return view! { <></> }.into_any();` early-return would change the inferred concrete type, breaking the `impl IntoView` return-type unification with the (unreachable) original body.
- **Fix:** Replaced the WHOLE body with `view! { <></> }` and `let _ = width; let _ = label;` to silence unused-param warnings while preserving the function signature so callers compile unchanged.
- **Files modified:** src/components/region/ornaments.rs (mutated then reverted)
- **Verification:** Cargo leptos watch rebuilt cleanly in 95s; both compile targets exit 0 post-revert.
- **Committed in:** `b6b381d` (Cycle 4 result captured in DIAGNOSIS.md)

**2. [Rule 3 - Blocking] Cycles 1-3 needed `#[allow(unreachable_code)]` annotation**
- **Found during:** Cycle 1 (Card mutation)
- **Issue:** Inserting `return view!{<></>}.into_any();` followed by the original `view!{...}.into_any()` produces an unreachable-code warning that with `-Dwarnings` (or close) would fail the build.
- **Fix:** Added `#[allow(unreachable_code)]` to the original branch body inside the mutation. The annotation is mechanical and removed on revert.
- **Files modified:** src/components/region/layout.rs (×2 cycles), src/components/region/typography.rs (mutated then reverted)
- **Verification:** All builds succeeded with the annotation; no warnings printed.
- **Committed in:** `b6b381d` (mutation pattern documented in DIAGNOSIS.md cycle subsections)

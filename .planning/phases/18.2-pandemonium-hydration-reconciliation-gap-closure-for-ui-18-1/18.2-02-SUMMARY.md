**2. [Rule 3 - Blocking] Cycle 5 mutation required `else if false` interstitial branch to avoid `unreachable_code` on the trailing-`if/else`-expression return type**
- **Found during:** Cycle 5 (CompanionSigil short-circuit)
- **Issue:** CompanionSigil's component body is the trailing expression `if is_pandemonium { ... .into_any() } else { ... .into_any() }` (NOT wrapped in a `move ||` closure). A naive `return view! { <></> }.into_any();` inside the `if` arm followed by the original `view! { <span ...> }.into_any()` would fail to compile because the `return` is from the enclosing component function (which has return type `impl IntoView`, NOT `AnyView`), so the `view! { <></> }.into_any()` AnyView would need to unify with `impl IntoView`. Even with `#[allow(unreachable_code)]`, type inference would still fail.
- **Fix:** Used an `else if false` interstitial: the original `<span>` arm was kept under an `else if false` branch so the unification still typechecks naturally. Pattern: `if is_pandemonium { view! { <></> }.into_any() } else if false { /* original span arm */ } else { /* Demacia arm */ }`. The `else if false` is dead at compile time but satisfies the AnyView unification.
- **Files modified:** `src/components/region/ornaments.rs` (mutated then reverted)
- **Verification:** Cargo leptos watch rebuilt cleanly in 50-75s; cycle ran to completion with a definitive culprit signal. Documented as the canonical Cycle-5 diff pattern in DIAGNOSIS.md.

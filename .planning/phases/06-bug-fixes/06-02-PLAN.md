**BUG-01 — Tree drafter branch does not freeze (per D: BUG-01):**

In `tree_drafter.rs`, the `do_branch_from` callback (around line 599-655) calls `select_node` inside a `spawn_local` async block. The fix per research:

1. Find the `spawn_local` block inside `do_branch_from` where `nodes_resource.refetch()` is called followed by `select_node(&new_node)`
2. Before `select_node(&new_node)`, explicitly:
   - Cancel any pending auto-save timer: read the timer signal and call `window.clear_timeout_with_handle(id)` if present (same pattern as the existing `cancel_autosave_timer` logic in the file)
   - Set `suppress_autosave.set(true)` to prevent the auto-save Effect from firing during signal batch updates
3. `select_node` internally also sets `suppress_autosave(true)` and schedules `setTimeout(0)` to re-enable it — this is correct and sufficient for re-enabling. The key fix is ensuring the timer is cancelled and suppression is active BEFORE `select_node` runs, closing the race window between `refetch()` completion and `select_node` execution.
4. If the file has a `cancel_autosave_timer` function/closure, call it. If not, inline the cancellation: read the timer `RwSignal<Option<i32>>`, call `clear_timeout_with_handle`, set to `None`.

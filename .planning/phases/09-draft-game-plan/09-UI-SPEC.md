## Spacing Scale

All spacing uses Tailwind v4 utility classes. The project's implicit scale follows 4-point increments.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, tight inline padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, champion grid gaps |
| md | 16px (`gap-4`, `p-4`) | Default element spacing, panel padding |
| lg | 24px (`gap-6`, `p-6`) | Section padding, sidebar inner padding |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (rarely needed) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Intel sidebar container: `p-4` (16px) with `gap-3` (12px) between elements — existing pattern, retain as-is (source: `draft.rs:1970`)
- Tab buttons: `px-2 py-1.5` — existing pattern, retain as-is (source: `draft.rs:1987`)
- Touch targets for buttons: minimum 32px height (existing `py-1.5` on tab buttons + `py-2` on dropdown items)

## Typography

All type uses Tailwind utility classes. The project uses `text-xs`, `text-sm`, `text-base` consistently.

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Label / metadata | 12px | 500 (medium) | 1.5 | `text-xs font-medium` |
| Body / inputs | 14px | 400 (regular) | 1.5 | `text-sm` |
| UI element headings | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` |
| Section headings | 16px | 600 (semibold) | 1.2 | `text-base font-semibold` |

Source: existing patterns in `draft.rs` intel sidebar and `champion_autocomplete.rs`.

**Permitted weights:** regular (400) and semibold (600) only. `font-medium` (500) is used only for tab buttons and small labels — existing convention, retain.

## Color

All colors use semantic CSS custom property tokens defined in `input.css`. Never use raw hex or Tailwind palette colors except for side-specific blues/reds (existing exception in CLAUDE.md).

| Role | Semantic Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `bg-base` | Page background |
| Secondary (30%) | `bg-surface`, `bg-elevated` | Cards, intel sidebar, dropdowns |
| Accent (10%) | `bg-accent` / `text-accent` / `border-accent` | Active tab button bg; active tab text; focused input border; selected sub-tab indicator |
| Destructive | `bg-red-700 text-white` (exception) | No destructive actions in this phase |

**Accent reserved for (explicit list):**
1. Active intel sidebar tab button background (`bg-accent text-accent-contrast`)
2. Active Notes sub-tab indicator (bottom border or background — new in this phase)
3. Focused state on opponent autocomplete input (`focus:border-accent/50`) — mirrors existing `ChampionAutocomplete` pattern
4. "Add New Opponent" button hover state (`hover:text-accent`)

**Side colors (existing exceptions — retain):**
- Blue side: `bg-blue-500 text-white`
- Red side: `bg-red-500 text-white`
- These are not semantic tokens; keep as-is per CLAUDE.md note on colored buttons

**Text tokens:**
- Primary text: `text-primary`
- Secondary/label text: `text-secondary`
- Muted/placeholder text: `text-muted`
- Dimmed/empty states: `text-dimmed`
- Error text: `text-red-400`

**Border tokens:**
- Dividers: `border-divider`
- Input borders: `border-outline` / `border-outline/50`

## Component Inventory

New UI elements introduced in this phase. All reuse existing Tailwind patterns from the codebase.

### 1. Opponent Autocomplete (header replacement for free-text input)

Pattern: Reuse `ChampionAutocomplete` structure with `Vec<Opponent>` instead of `Vec<Champion>`.

```
[  Search opponents...  ▾ ]  [ + Add New ]
[─────────────────────────]
[ Team Evil                ]   ← bg-elevated, border-divider dropdown
[ Team Alpha               ]
```

- Container: `relative` positioned wrapper
- Input: `w-full bg-surface/50 border border-outline/50 rounded-lg px-3 py-2 text-primary text-sm placeholder-dimmed focus:outline-none focus:border-accent/50 transition-colors`
- Dropdown: `absolute z-50 mt-1 w-full bg-elevated border border-divider rounded-lg shadow-xl overflow-hidden max-h-56 overflow-y-auto`
- Dropdown item: `w-full flex items-center gap-2 px-3 py-2 hover:bg-overlay transition-colors text-left cursor-pointer text-primary text-sm`
- "Add New Opponent" button: `px-3 py-2 rounded-lg bg-overlay hover:bg-overlay-strong text-secondary hover:text-primary text-sm transition-colors cursor-pointer flex-shrink-0`

### 2. Notes Tab (4th intel sidebar tab)

Tab button label: "Notes" — added to existing `["pools", "their_picks", "matchups"]` array.
Active state: `bg-accent text-accent-contrast` (matches existing tab pattern).
Inactive state: `bg-overlay text-muted hover:bg-overlay-strong` (matches existing tab pattern).

### 3. Notes Champion Sub-tabs

Second row of tabs inside the Notes tab — one per our-side picked champion.

```
[ Jinx ]  [ Orianna ]  [ Renekton ]
```

- Sub-tab row: `flex gap-1 flex-wrap border-b border-divider pb-2`
- Active sub-tab: `px-2 py-1 rounded-t text-xs font-medium border-b-2 border-accent text-primary bg-transparent`
- Inactive sub-tab: `px-2 py-1 rounded-t text-xs font-medium text-muted hover:text-secondary transition-colors cursor-pointer`
- Champion portrait (optional, Claude's discretion): 20×20px (`w-5 h-5`) rounded Community Dragon icon inline before name

### 4. Collapsible Note Type Section

One section per note type (matchup, power_spike, combo, lesson, synergy, positioning).

```
▼ Matchups  ────────────────────
  vs Caitlyn: Watch laning phase
  vs Jinx: Push for level 2 ...

▶ Power Spikes  ─────────────────
  (collapsed)
```

- Section header: `flex items-center justify-between py-2 cursor-pointer hover:text-primary transition-colors`
- Section label: `text-xs font-medium text-secondary uppercase tracking-wide`
- Toggle icon: inline chevron SVG, 12×12px, rotates 90° when expanded (`transition-transform`)
- Section body: `flex flex-col gap-1 pb-2`
- Note item: `text-xs text-muted leading-relaxed`
- Matchup sections: expanded by default; all other types: collapsed by default

### 5. Notes Empty State (per champion sub-tab)

```
No notes for Jinx yet.
Add notes in Champion Pool →
```

- Container: `flex flex-col items-center gap-2 py-6 text-center`
- Heading: `text-sm text-dimmed`
- Link: `text-xs text-accent hover:underline cursor-pointer` — navigates to `/champion-pool`

### 6. Back Button (Intel Panel Matchups tab)

Appears in champion detail view. Sets `matchup_champion` to `None`.

```
← Back to champions
```

- Style: `flex items-center gap-1 text-xs text-muted hover:text-secondary transition-colors cursor-pointer mb-2`
- Chevron: inline `<` SVG, 12×12px

### 7. Game Plan Prefill Fields (existing page, extended)

No new UI elements — extends existing `game_plan.rs` form fields. The prefill Effect pre-populates:
- Plan name input (`plan_name`): pre-filled as `"<draft name> plan"` — existing `<input>` element
- Strategy tag select (`win_condition_tag`): pre-selected from most common draft tag — existing `<select>` element
- Per-role strategy textareas: pre-filled with `"<ChampionName>: "` prompt — existing `<textarea>` elements

No visual changes to the game plan page — autofill is a data behavior change only.

## Interaction Contracts

### Opponent Dropdown (DRFT-01)

1. On focus/input: show filtered `opponents_list` in dropdown (max 8 results, matches by substring on opponent name)
2. On item select (mousedown): set `opponent` signal to opponent record ID; set `filter_text` to opponent display name; close dropdown; set `selected_opponent_id` to same ID; set `intel_open` to `true` (D-04)
3. On blur: close dropdown after 150ms delay (same pattern as `ChampionAutocomplete`)
4. Backward compat: when draft loads and stored `opponent` value doesn't match any known opponent ID, display the raw string as filter text; do not set `selected_opponent_id`

### "Add New Opponent" Button (DRFT-02)

1. On click: capture all draft signal values eagerly (CLAUDE.md rule 54)
2. If `draft_name` is empty: show toast "Give this draft a name first — notes weren't saved." and abort navigation
3. If `draft_name` is non-empty: call `save_draft` or `update_draft` via `spawn_local`; on success, navigate to `/opponents?return_to=draft&draft_id={id}` via `window.location().set_href()`
4. Opponents page: reads `?return_to=draft&draft_id=` query param; renders "Back to Draft" link when param is present

### Matchups Tab Ban Filtering (DRFT-03)

1. Champion selection grid in Matchups tab: filters out champions present in ban slots (indices 0–5, 12–15 per `slot_meta()`)
2. Filter is reactive — updates immediately when bans change
3. No visual indication that a champion is "filtered" — banned champions simply do not appear in the list

### Back Button in Matchups Tab (DRFT-04)

1. Back button visible when `matchup_champion` is `Some(_)` (detail view is open)
2. On click: `set_matchup_champion.set(None)` — returns to champion selection list
3. No animation — immediate state change (matches existing Leptos pattern)

### Notes Tab Auto-open (DRFT-05 / D-13)

1. `pool_notes` Resource keyed on `our_picks_for_notes` (derived from `draft_slots` + `our_side`)
2. Effect watches `pool_notes` resource: when a new pick arrives with non-empty notes, set `intel_open.set(true)`, `intel_tab.set("notes")`, `notes_champion_tab.set(champion_name)`
3. Auto-switch only fires if user has not manually selected a different notes sub-tab for that champion (avoid overriding manual navigation)
4. Note type sections: matchup type expanded by default; power_spike, combo, lesson, synergy, positioning collapsed by default
5. `collapsed_note_types` is a `RwSignal<Vec<String>>` — resets when `notes_champion_tab` changes

### Game Plan Autofill (PLAN-01, PLAN-03)

1. Prefill Effect in `game_plan.rs` fires when `prefill_data` Resource resolves to `Some(draft)`
2. Sets `plan_name` to `"{draft.name} plan"`
3. Sets `win_condition_tag` to most-common tag (first in list on tie, per D-17)
4. Sets per-role strategy fields to `"{champion_name}: "` where role assignment exists in `DraftAction.role`; leaves unassigned roles as empty string
5. All prefilled fields remain fully editable (D-20)
6. `clear_editor` callback resets `plan_name` to empty string alongside existing field resets

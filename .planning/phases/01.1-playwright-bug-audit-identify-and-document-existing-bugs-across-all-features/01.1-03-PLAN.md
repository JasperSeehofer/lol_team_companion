**Step 1: Run the full e2e suite and capture output.**

```bash
cd e2e && npx playwright test --project=chromium 2>&1
```

Save the full output for analysis. Note every test that failed and why.

**Step 2: For each failing test, determine if it's a bug or a test issue.**

- If the failure reproduces a real application bug (server error, UI not updating, WASM panic, missing data): document it as a bug.
- If the failure is a test issue (wrong selector, timing too tight, test logic error): fix the test and re-run.

**Step 3: Create BUG-REPORT.md** in the phase directory with this structure:

```markdown
# Bug Report: Phase 01.1 Playwright Audit

**Date:** [today]
**Audited by:** Automated Playwright e2e suite
**Server:** cargo leptos watch (dev mode)
**Browser:** Chromium (Playwright)

## Summary

- **Total bugs found:** N
- **Known bugs confirmed:** 2 (draft list, tree drafter freeze)
- **New bugs discovered:** M
- **Routes fully passing:** K/14

## Bugs

### BUG-001: [Title]
- **Severity:** critical | major | minor | cosmetic
- **Route:** /path
- **Test file:** e2e/tests/audit-xxx.spec.ts
- **Test name:** "test name here"
- **Steps to reproduce:**
  1. Step 1
  2. Step 2
- **Expected:** What should happen
- **Actual:** What happens instead
- **Suspected cause:** (if identifiable from research or code review)
- **Console errors:** (if any captured)

[Repeat for each bug]

## Routes Status

| Route | Status | Notes |
|-------|--------|-------|
| /draft | BUG | BUG-001: saved draft not in list |
| /tree-drafter | BUG | BUG-002: unresponsive on switch |
| /game-plan | OK/BUG | ... |
| ... | ... | ... |

## Test Suite Status

| Test File | Total | Pass | Fail (expected) | Fail (unexpected) |
|-----------|-------|------|------------------|-------------------|
| audit-draft.spec.ts | 5 | N | M | 0 |
| ... | ... | ... | ... | ... |
```

**Step 4: Annotate all confirmed bug tests with test.fail().**

For each test that fails due to a confirmed application bug:
1. Add `test.fail()` as the first line inside the test body (NOT in the test name string — use the Playwright API)
2. Add a comment: `// BUG-NNN: [brief description] — see BUG-REPORT.md`

Do NOT add test.fail() to tests that fail due to test issues — fix those instead.

**Step 5: Re-run the full suite to confirm it's green.**

```bash
cd e2e && npx playwright test --project=chromium
```

All tests should now pass (test.fail() inverts expected failures).

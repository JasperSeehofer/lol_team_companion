# Phase 10 — UI Design Contract

> Visual and interaction contract for Phase 10: Opponents Redesign.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom Tailwind v4 semantic tokens |
| Preset | not applicable |
| Component library | none — Leptos components with Tailwind utility classes |
| Icon library | Unicode characters + inline SVG (existing pattern) |
| Font | System UI stack (Tailwind default) |

**Source:** `input.css` — Tailwind v4 `@theme` block with full semantic token set. No shadcn. Project is Leptos/Rust, not React.

**Existing semantic tokens (use these, never hardcode):**
- Backgrounds: `bg-base`, `bg-surface`, `bg-elevated`, `bg-overlay`, `bg-overlay-strong`
- Text: `text-primary`, `text-secondary`, `text-muted`, `text-dimmed`
- Borders: `border-divider`, `border-outline`
- Accent: `bg-accent`, `hover:bg-accent-hover`, `text-accent`, `text-accent-contrast`

**Exceptions to semantic tokens (from CLAUDE.md):**
- OTP warning badge: `text-orange-400` — colored badges with literal color keep literal class
- Stale recency indicator: `text-orange-400` — same exception
- Fetch success indicator: `text-green-400`
- Fetch error indicator: `text-red-400`
- Delete/destructive buttons: `bg-red-700 text-white` or `text-red-400`

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding (`px-1`), inline gaps between badge elements |
| sm | 8px | Gap between role-row label and input, gap between champion pills |
| md | 16px | Card internal padding (`p-4`), gap between cards in role form |
| lg | 24px | Section padding (`p-6`), gap between page sections |
| xl | 32px | Layout column gap (`gap-6` ≈ 24px is standard; page-level margin `py-8` = 32px) |
| 2xl | 48px | Major section breaks (page top padding) |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Touch targets for icon buttons (refresh, collapse toggle): minimum 32px × 32px (`w-8 h-8`) to ensure clickability
- Champion pills use compact padding: `px-2 py-0.5` (8px × 2px) — existing project pattern

**Source:** Existing `opponents.rs` pattern: `px-4 py-3`, `p-4`, `gap-2`, `gap-6`. Confirmed from CONTEXT.md and RESEARCH.md.

## Typography

| Role | Size | Weight | Line Height | Token/Class |
|------|------|--------|-------------|-------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label / badge | 12px | 400 (regular) | 1.4 | `text-xs` |
| Section heading | 16px | 600 (semibold) | 1.2 | `text-base font-semibold` |
| Page heading | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

**Notes:**
- Body (`text-sm` / 14px) is the dominant size for all form inputs, player names, and champion pills — consistent with existing opponents.rs usage
- Label (`text-xs` / 12px) for all badges: OTP badge, mastery level suffix, recency timestamp, pool analysis metrics
- Section headings (`text-base font-semibold`) for card header role labels and panel section titles
- Page heading (`text-2xl font-bold`) for the `<h1>"Opponents"</h1>` — preserve existing

**Source:** Existing `opponents.rs` typography: `text-sm`, `text-xs`, `text-2xl font-bold`, `text-sm font-semibold`. Pre-populated from codebase.

## Color

| Role | Semantic Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `bg-base` | Page background |
| Secondary (30%) | `bg-surface`, `bg-elevated` | Opponent list panel, player cards, detail panel |
| Accent (10%) | `bg-accent` / `text-accent` | Primary CTAs only (see reserved list below) |
| Destructive | `text-red-400` / `bg-red-700 text-white` | Delete opponent confirmation button only |
| Warning (status) | `text-orange-400` | OTP badge text, stale recency indicator text |
| Success (status) | `text-green-400` | Per-player fetch success check icon |
| Error (status) | `text-red-400` | Per-player fetch error message text |

**Accent reserved for:**
1. "Save & Fetch" primary CTA button (`bg-accent text-accent-contrast`)
2. "Refresh All" team-level button (`bg-accent text-accent-contrast`)
3. Selected opponent row highlight border or background tint in the list panel
4. Back-to-draft navigation link (`text-accent hover:underline`)

**Accent NOT used for:**
- Individual player refresh icon buttons (use `text-muted hover:text-secondary`)
- Collapse/expand toggle (use `text-muted`)
- Cancel / secondary buttons (use `bg-elevated border-divider text-secondary`)
- Role labels (use `text-secondary`)
- Champion pills (use `bg-surface border-divider/50 text-secondary`)

**OTP badge color:** `bg-orange-500/20 text-orange-400 border border-orange-500/30` — warning palette, not accent. Triggered when >60% of recent games on single champion.

**Mastery badge color:** Mastery level indicator uses muted text within the champion pill — no separate color. Format: `{Champion} M{level}` in `text-xs text-secondary`.

**Stale recency:** `text-orange-400` when `last_fetched` age >= 7 days. `text-muted` when fresh (< 7 days). No recency info shows `text-dimmed` with "Never fetched".

**Source:** CONTEXT.md D-06 (orange OTP badge), D-08 (stale-orange after 7+ days). Existing `input.css` token set. Existing opponents.rs card pattern.

```
  │     │ [Role icon] Support [Name#Tag input   ] │

  └── [Cancel] [Save & Fetch] buttons (right-aligned)
```

Role label width: fixed `w-20` to align all inputs. Role label: `text-sm text-secondary font-medium`.

### Opponent Detail Panel (existing opponent — edit mode)

### PlayerCard Layout
```
bg-elevated rounded-lg border border-divider/50 p-4
├── Row 1 (card header):
│     [Role icon 20px]  [Summoner name text-sm font-medium]
```

## Interaction Contract

### 5-Role Form — Input Validation
- Each Riot ID field validates on submit (not on blur) for `#` separator presence
- If `#` is absent and field is non-empty: inline error `text-xs text-red-400` below the row — "Use Name#Tag format (e.g. Faker#KR1)"
- Empty fields pass validation without error (D-03: partial entry allowed)
- Team name must be non-empty — inline error "Team name is required" if blank on submit

### Save & Fetch Button States
| State | Visual |
|-------|--------|
| Idle | `bg-accent text-accent-contrast` — "Save & Fetch" |
| Saving | Disabled, `opacity-50 cursor-not-allowed` — "Saving..." |
| Fetching | Enabled (per-player status shows progress) — label returns to "Save & Fetch" |
| Error | Toast: error message. Button re-enabled |

### Per-Player Fetch Status Icons
| State | Visual |
|-------|--------|
| `Idle` | No icon shown |
| `Fetching` | Animated spinner SVG `animate-spin w-4 h-4 text-muted` |
| `Success` | Checkmark `text-green-400 w-4 h-4` |
| `Error(msg)` | Red X `text-red-400 w-4 h-4` + tooltip or inline `text-xs text-red-400` error text below input |

### Collapsible Pool Analysis
- Toggle button: `text-xs text-muted hover:text-secondary cursor-pointer flex items-center gap-1`
- Collapsed: shows "Pool Analysis ▸" (right-pointing triangle)
- Expanded: shows "Pool Analysis ▾" (down-pointing triangle) + content
- Transition: no animation required (CONTEXT.md: Claude's discretion — use no-animation for simplicity)
- State: `RwSignal<bool>` per player card, initialized `false`

### Refresh All Button
- Location: opponent detail panel header, right side
- Appearance: `bg-elevated border border-divider text-secondary text-sm px-3 py-1.5 rounded-lg hover:bg-overlay` (secondary style, not accent — accent is reserved for Save & Fetch)
- While fetching: disabled, shows "Refreshing..." text
- After completion: toast "All players refreshed" or "X/5 players refreshed" if partial failures

### Individual Refresh Button
- Small icon button: `w-8 h-8 flex items-center justify-center rounded text-muted hover:text-secondary hover:bg-elevated transition-colors cursor-pointer`
- Icon: refresh/reload unicode or simple SVG (2 arrows in circle)
- While fetching: replaces icon with spinner

### Delete Opponent
- Trigger: "Delete" button in detail panel header (`text-red-400 text-sm hover:text-red-300`)
- Confirmation: inline confirm state replaces the button with "Confirm delete?" + [Cancel] [Delete] buttons — same pattern as existing `confirm_delete: RwSignal<bool>` in opponents.rs
- Confirmation Delete button: `bg-red-700 text-white text-sm px-3 py-1.5 rounded-lg hover:bg-red-600`
- No modal overlay — inline confirmation is the established project pattern

## Status States per Player Card

Every `PlayerCard` must handle these states visually:

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| No Riot ID | `riot_summoner_name` is None/empty | Input shows placeholder, no recency badge, no champion pills, no pool analysis |
| Has Riot ID, never fetched | `last_fetched` is None | Riot ID shown in input, "Never fetched" text-dimmed badge, no champion pills |
| Has data, fresh | `last_fetched` < 7 days | Recency badge text-muted, champion pills shown, pool analysis available |
| Has data, stale | `last_fetched` >= 7 days | Recency badge `text-orange-400`, champion pills shown, pool analysis available |
| Fetching | `FetchState::Fetching` | Spinner in input row, all other content visible (don't hide existing data) |
| Fetch success | `FetchState::Success` | Green checkmark briefly, then transitions back to Idle after 3 seconds |
| Fetch error | `FetchState::Error(msg)` | Red X + error message text-xs text-red-400 below input |
| OTP detected | `otp_champion` is Some | Orange OTP badge in card header next to role label |

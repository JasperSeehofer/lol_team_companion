## Spacing Scale

Declared values (multiples of 4 only). Using Tailwind v4 defaults mapped to 8-point scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1) | Icon gaps, badge padding, tight inline spacing |
| sm | 8px (p-2) | Compact element padding, tag pill spacing |
| md | 16px (p-4) | Default card padding, form field spacing |
| lg | 24px (p-6) | Section-level padding, card group gaps |
| xl | 32px (p-8) | Page horizontal padding, major layout gaps |
| 2xl | 48px (py-12) | Page top padding, between major sections |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Accordion expanded row: `p-4` (16px) padding inside the expansion cell
- Strategy tag cards: `p-4` (16px) consistent with `StatCard` pattern (source: `stat_card.rs` line 10)
- Table rows: `px-4 py-3` (16px/12px) for comfortable row height without waste

**Source:** Confirmed from existing patterns in `post_game.rs`, `game_plan.rs`, `stat_card.rs`.

## Color

All values are semantic token references from `input.css`. Dark theme is default; light theme is opt-in via `data-theme="light"`.

| Role | Semantic Token | Resolved (dark) | Usage |
|------|---------------|-----------------|-------|
| Dominant (60%) | `bg-base` | oklch(0.13) ≈ gray-950 | Page background |
| Secondary (30%) | `bg-surface`, `bg-elevated` | gray-900, gray-800 | Cards, nav, expanded accordion rows |
| Accent (10%) | `bg-accent` / `text-accent` | yellow-400 | Reserved elements only (see below) |
| Destructive | `bg-red-700 text-white` (literal) | red-700 | No destructive actions in this phase |
| Win indicator | `text-emerald-400` (literal) | emerald-400 | Win outcome badges, win count cells |
| Loss indicator | `text-red-400` (literal) | red-400 | Loss outcome badges, loss count cells |

**Accent reserved for:**
1. Primary CTA button ("Add Review", "Save Review") — `bg-accent text-accent-contrast`
2. Active strategy tag pill (selected state) — `bg-accent text-accent-contrast`
3. Currently active accordion row indicator — `border-l-2 border-accent`
4. Sortable column header active sort direction indicator — `text-accent`
5. Star rating fill color — `text-accent` for filled stars

**Accent is NOT used for:** table row hover, win/loss badges, strategy tag card backgrounds, inline links.

**Strategy tag card colors** (Claude's discretion, confirmed here):

#### 4. Star Rating Input (post_game.rs form additions)

1-5 interactive star selector for rating field on post-game review form.

**Display:** Unicode `★` (U+2605 filled) and `☆` (U+2606 empty). Five stars, horizontally arranged.
**Interaction:** Click on star N sets rating to N. Hover previews the selected state.
**Size:** `text-lg` (18px) for input stars, `text-sm` (14px) for read-only display in table.
**Selected state:** `text-accent` (yellow). **Unselected:** `text-dimmed`. **Hover preview:** `text-accent/70`.

```
<div class="flex gap-1">
  {(1..=5).map(|n| view! {
    <button
      class=move || if rating.get().map_or(false, |r| r >= n) {
        "text-accent text-lg hover:text-accent-hover transition-colors"
      } else {
        "text-dimmed text-lg hover:text-accent/70 transition-colors"
      }
      on:click=move |_| set_rating.set(Some(n))
    >
      {if filled { "★" } else { "☆" }}
    </button>
  }).collect_view()}
</div>
```

#### 5. Win/Loss Outcome Selector (post_game.rs form additions)

Three-state toggle: None (default) / Win / Loss. Shown in post-game review creation form.

```
<div class="flex gap-2">
```

Button size: `px-3 py-1.5 rounded-lg text-sm font-medium transition-colors`.

## Interaction Contracts

### Sortable Table

- Click a sortable column header → toggle ASC/DESC on that column; other columns reset to neutral.
- Active sort column header: `text-primary` (was `text-muted`), plus sort direction indicator: `↑` (ASC) or `↓` (DESC) in `text-accent`.
- Sort state: `RwSignal<(SortColumn, SortDir)>` in the component, never persisted to URL (no filters in v1.1).
- Default sort: W-L record descending (most wins first).

### Accordion

- Click a plan row → expands inline reviews. Clicking the same row again collapses it.
- Only one row open at a time — opening a new row closes the previous.
- No animation in v1.1 (deferred to future). Content appears/disappears instantly.
- Expanded row has `border-l-2 border-accent` left indicator per review item.
- The expand chevron indicator: `▼` when open, `▶` when closed, `text-muted text-xs` appended to Plan name cell.

### Empty States

**Analytics page — no team:**
```
<div class="text-center py-16">
  <p class="text-dimmed text-sm mb-3">No analytics data yet</p>
  <p class="text-dimmed text-xs mb-4">Create or join a team to get started.</p>
  <a href="/team/roster" class="bg-accent hover:bg-accent-hover text-accent-contrast font-bold rounded px-3 py-1.5 text-xs transition-colors">
    Join a team
  </a>
</div>
```

**Analytics page — team exists but no reviews with game plans linked:**
```
<div class="text-center py-16">
  <p class="text-dimmed text-sm mb-3">No plan effectiveness data yet</p>
  <p class="text-dimmed text-xs">Link post-game reviews to game plans to start tracking strategy outcomes.</p>
</div>
```

**Strategy tag cards — tag has zero reviews:**
Do not render a card for tags with 0 games. Only render cards for tags with at least 1 linked review.

## Implementation Notes for Executor

1. **No new color tokens needed.** All colors in this phase use existing semantic tokens (`bg-accent`, `text-emerald-400`, `text-red-400`) or existing Tailwind opacity modifiers. Do not add to `input.css`.

2. **`stat_card.rs` reused as-is.** Strategy tag summary cards follow the same structure — `bg-elevated border border-divider rounded-lg p-4`. The tag-specific color tints are applied additionally, not replacing the base card styles.

3. **Accordion uses `into_any()`.** The expanded and collapsed row views are structurally different. Both arms must call `.into_any()` per CLAUDE.md rule 19.

4. **Win/Loss color exceptions.** `text-emerald-400` and `text-red-400` are used literally (not via semantic tokens) for win/loss indicators — same pattern as `text-white` on colored buttons per CLAUDE.md code style section.

5. **Star rating read-only display.** In the table, stars are `text-sm` static text. In the form, they are `text-lg` interactive buttons. Use `collect_view()` on the iterator per CLAUDE.md rule 25.

6. **"Fetch result" button.** Matches the inline action button pattern established in `post_game.rs:610,625` — `inline-flex items-center gap-1 bg-surface border border-outline/50 text-muted text-xs rounded px-2 py-1 hover:text-primary hover:border-accent/50 transition-colors`.

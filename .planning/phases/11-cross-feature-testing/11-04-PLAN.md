The seed binary structure:

```rust
//! Seed binary: populates the app with realistic test data.
//!
//! Run with: cargo run --features ssr --bin seed
//! Requires: RIOT_API_KEY in .env (optional — falls back to static data)
//! IMPORTANT: Stop the dev server first (SurrealKV exclusive lock)

use std::env;

#[tokio::main]
async fn main() {
    dotenvy::dotenv().ok();

    let dry_run = env::args().any(|a| a == "--dry-run");
    let data_dir = env::var("SURREAL_DATA_DIR").unwrap_or_else(|_| "./data".to_string());

    println!("Seed: initializing database from {data_dir}...");
    let db = lol_team_companion::server::db::init_db(&data_dir)
        .await
        .expect("DB init failed — is the dev server still running? Stop it first.");

    if dry_run {
        println!("[DRY RUN] Would clean up seed data and create 2 teams with 5 users each.");
        return;
    }

    // Phase 1: Clean up existing seed data
    cleanup_seed_data(&db).await;

    // Phase 2: Create Team A (5 users)
    let team_a_id = create_seed_team(&db, "Alpha Wolves", &[
        ("seed-alpha-top", "SeedTop1", "EUW"),
        ("seed-alpha-jg", "SeedJungle1", "EUW"),
        ("seed-alpha-mid", "SeedMid1", "EUW"),
        ("seed-alpha-bot", "SeedBot1", "EUW"),
        ("seed-alpha-sup", "SeedSup1", "EUW"),
    ]).await;

    // Phase 3: Create Team B (5 users)
    let team_b_id = create_seed_team(&db, "Beta Dragons", &[
        ("seed-beta-top", "SeedTop2", "EUW"),
        ("seed-beta-jg", "SeedJungle2", "EUW"),
        ("seed-beta-mid", "SeedMid2", "EUW"),
        ("seed-beta-bot", "SeedBot2", "EUW"),
        ("seed-beta-sup", "SeedSup2", "EUW"),
    ]).await;

    // Phase 4: Populate data for each team
    populate_team_data(&db, &team_a_id, "Alpha Wolves").await;
    populate_team_data(&db, &team_b_id, "Beta Dragons").await;

    // Phase 5: Create opponent scouting profiles
    create_opponent_profiles(&db, &team_a_id).await;

    println!("\nSeed complete! Created 2 teams with full demo data.");
    println!("Login with any seed account: seed-alpha-top@example.com / seedpass123");
}
```

The seed account email pattern: `{username}@example.com` (e.g., `seed-alpha-top@example.com`). Password for all seed accounts: `seedpass123`.

**`cleanup_seed_data`** function: Delete in reverse dependency order to avoid orphans:
```rust
async fn cleanup_seed_data(db: &Surreal<Db>) {
    println!("Cleaning up existing seed data...");
    let queries = [
        "DELETE post_game_learning WHERE created_by IN (SELECT id FROM user WHERE string::starts_with(email, 'seed-'))",
        "DELETE game_plan WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta'))",
        "DELETE draft_action WHERE draft IN (SELECT id FROM draft WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta')))",
        "DELETE draft WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta'))",
        "DELETE champion_note WHERE user IN (SELECT id FROM user WHERE string::starts_with(email, 'seed-'))",
        "DELETE champion_pool WHERE user IN (SELECT id FROM user WHERE string::starts_with(email, 'seed-'))",
        "DELETE team_member WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta'))",
        "DELETE opponent_player WHERE opponent IN (SELECT id FROM opponent WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta')))",
        "DELETE opponent WHERE team IN (SELECT id FROM team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta'))",
        "DELETE team WHERE string::starts_with(name, 'Alpha') OR string::starts_with(name, 'Beta')",
        "DELETE user WHERE string::starts_with(email, 'seed-')",
    ];
    for q in queries {
        if let Err(e) = db.query(q).await {
            eprintln!("  Warning during cleanup: {e}");
        }
    }
    println!("  Cleanup done.");
}
```

**`create_seed_team`** function: Creates users with hashed passwords, creates team, adds members.

Uses `lol_team_companion::server::auth::hash_password()` for password hashing.
Uses raw SurrealQL queries for user creation (since there may not be a `create_user` DB function — check `db.rs`).

Each user: email = `{username}@example.com`, password = hashed `seedpass123`.

Optionally link Riot accounts: If `RIOT_API_KEY` is set, call `riot::get_puuid()` for each user's Riot ID. If not set or if the call fails, skip PUUID linking and print a warning.

Add `tokio::time::sleep(Duration::from_millis(150))` between Riot API calls to respect rate limits (per research -- 20 req/1s limit).

**`populate_team_data`** function: For each team, create:
- Champion pool entries: Use a static list of champion names per role (e.g., top: Darius, Garen, Camille, Fiora, Aatrox). 3 tiers: "S", "A", "B". 5 champions per team member.
- Champion notes: 1-2 notes per champion using hardcoded text (e.g., "Strong level 6 all-in", "Weak vs poke lanes").
- 2-3 drafts: Each with 10 pick + 10 ban DraftActions. Use static champion lists. Set team side to "blue".
- 1-2 game plans: Linked to drafts via `draft_id`. Set `win_condition_tag` to realistic values ("teamfight", "split-push", "engage").
- 1-2 post-game reviews: Linked to game plans via `game_plan_id`. Set `win_loss` to `Some("win".into())` or `Some("loss".into())` and `rating` to `Some(3)` or `Some(4)` to populate the analytics page.

If `RIOT_API_KEY` is available, optionally fetch champion mastery data and use it to populate champion pool entries with real mastery levels. If not, use static data.

**`create_opponent_profiles`** function: Create 1 opponent with 5 players (one per role). Use static Riot IDs or skip if no API key.

All DB calls should use `.await?.check()?` for writes (per CLAUDE.md rule 27) or handle errors gracefully with `eprintln!` + continue (seed should be tolerant of individual failures).

Print progress to stdout: `"  Created user {email}"`, `"  Created team {name}"`, etc.

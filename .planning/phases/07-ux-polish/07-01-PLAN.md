  <action>
    **1. Create `src/models/utils.rs`** with a testable timestamp formatter:

    Create a public function `format_timestamp(s: &str) -> String` and an internal helper `format_timestamp_with_now(s: &str, now: DateTime&lt;Utc&gt;) -> String` for testability. The public function calls the internal one with `Utc::now()`.

    Implementation per D-01/D-02/D-03:
    - Parse input with `s.parse::&lt;DateTime&lt;Utc&gt;&gt;()`. On parse failure, return `s.to_string()`.
    - Compute age = `now.signed_duration_since(dt)`.
    - If age &lt; 0 (future): use absolute format.
    - If age &lt; 1 minute: return `"just now"`.
    - If age &lt; 1 hour: return `"{mins} minute(s) ago"` with proper pluralization.
    - If age &lt; 24 hours: return `"{hours} hour(s) ago"` with proper pluralization.
    - If same year as `now`: return `dt.format("%e %b, %H:%M")` trimmed (e.g. "19 Mar, 14:30"). Note: `%e` gives space-padded day; use `%-d` for unpadded or `.trim()` the result.
    - If different year: return `dt.format("%-d %b %Y, %H:%M")` (e.g. "5 Dec 2025, 14:30").

    Use `chrono::{DateTime, Utc, TimeDelta}`. `Duration` is deprecated in recent chrono; use `TimeDelta` if available, or `chrono::Duration` if the project's chrono 0.4 version predates `TimeDelta`. Check at compile time.

    Add `#[cfg(test)] mod tests` block with tests using `format_timestamp_with_now()` with fixed `now` values to ensure deterministic results. Cover all branches: just now, minutes ago, hours ago, same-year absolute, different-year absolute, parse failure, empty string, future timestamp.

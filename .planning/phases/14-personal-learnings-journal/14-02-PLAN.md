### Server Functions (5 total, defined before components per CLAUDE.md rule 34)

All server functions use `use_context::<Arc<Surreal<Db>>>()` for DB access, `leptos_axum::extract::<AuthSession>()` for auth, and return `ServerFnError`. Import server-only crates inside `#[server]` body (CLAUDE.md rule 9).

1. **`list_learnings() -> Result<Vec<PersonalLearning>, ServerFnError>`** — calls `db::list_personal_learnings(db, &user_id)`. Returns `Ok(vec![])` if no user (CLAUDE.md rule 44).

2. **`get_learning(id: String) -> Result<Option<PersonalLearning>, ServerFnError>`** — calls `db::get_personal_learning(db, &id)`. For edit mode.

3. **`save_learning(title: String, learning_type: String, champion: String, opponent: String, what_happened: String, what_i_learned: String, next_time: String, tags_json: String, win_loss: String, match_riot_id: String, game_timestamp_ms: String, event_name: String) -> Result<String, ServerFnError>`** — Deserialize `tags_json` as `Vec<String>` via `serde_json::from_str`. Convert empty strings to `None` for optional fields. Parse `game_timestamp_ms` string to `Option<i64>`. Build `PersonalLearning` struct, call `db::create_personal_learning`. Return created ID.

4. **`update_learning(id: String, title: String, learning_type: String, champion: String, opponent: String, what_happened: String, what_i_learned: String, next_time: String, tags_json: String, win_loss: String, match_riot_id: String, game_timestamp_ms: String, event_name: String) -> Result<(), ServerFnError>`** — Same field handling as save, but with `id` set. Calls `db::update_personal_learning`.

5. **`delete_learning(id: String) -> Result<(), ServerFnError>`** — calls `db::delete_personal_learning(db, &id)`.

### Browse Page Component: `PersonalLearningsPage`

Auth guard pattern: fetch `get_current_user()` server fn, redirect to `/auth/login` if None (`#[cfg(feature = "hydrate")]`).

**State signals:**
- `filter_type: RwSignal<String>` — "all" | "general" | "champion" | "matchup"
- `filter_champion: RwSignal<String>` — "all" | specific champion name
- `filter_tag: RwSignal<String>` — "all" | specific tag
- `sort_mode: RwSignal<String>` — "newest" | "champion"
- `expanded_id: RwSignal<Option<String>>` — currently expanded card ID

**Resource:** `let learnings = Resource::new(|| (), |_| list_learnings());`

**Layout** (per UI-SPEC page layout):
```
Page heading: <h1 class="text-3xl font-bold text-primary">"Personal Learnings"</h1>
Subtext: <p class="text-sm text-muted mt-1">"Your private post-game reflections"</p>
New button: <a href="/personal-learnings/new" class="bg-accent hover:bg-accent-hover text-accent-contrast font-bold px-5 py-2 rounded-lg text-sm transition-colors">"+ New Learning"</a>
```

**Filter bar** (per UI-SPEC "Filter Bar" interaction contract):
- Four `<select>` dropdowns in a horizontal flex row
- Style: `bg-surface/50 border border-outline/50 rounded-lg px-3 py-2 text-primary text-sm focus:outline-none focus:border-accent/50`
- Type dropdown: options "All", "General", "Champion", "Matchup"
- Champion dropdown: dynamically populated from unique champion values in loaded learnings
- Tag dropdown: populated from `LEARNING_TAGS` constant
- Sort dropdown: "Newest first", "By champion" — right-aligned with `ml-auto`
- All use `on:change` with `event_target_value(&ev)` to update their signal

**Card grid** (per UI-SPEC "Learning Card" interaction contract):
- Container: `<div class="grid grid-cols-2 lg:grid-cols-3 gap-6">`
- Client-side filtering inside `Suspense` `move ||` block:
  - Filter by type: `filter_type.get() == "all" || l.learning_type == filter_type.get()`
  - Filter by champion: `filter_champion.get() == "all" || l.champion.as_deref() == Some(filter_champion.get().as_str())`
  - Filter by tag: `filter_tag.get() == "all" || l.tags.contains(&filter_tag.get())`
- Sort: when `sort_mode.get() == "newest"`, keep default order (already ORDER BY created_at DESC). When `"champion"`, collect into `BTreeMap<String, Vec<PersonalLearning>>` keyed by `champion.clone().unwrap_or("General".into())`, render with section headers (`text-xs text-dimmed uppercase tracking-wider py-2`).

**Each card:**
- Default class: `bg-elevated/30 border border-divider/30 rounded-xl p-4 hover:bg-overlay/30 transition-all cursor-pointer`
- Expanded class: `bg-accent/10 border border-accent/30 rounded-xl p-4 transition-all`
- Toggle: `on:click` with `get_untracked()` on `expanded_id`, toggle logic (if same ID, set None; else set Some(id))
- Card content (collapsed): type badge (`bg-overlay text-muted text-xs font-bold rounded px-2 py-0.5`), win/loss badge (Win: `bg-emerald-500/20 text-emerald-400 border border-emerald-500/30`; Loss: `bg-red-500/10 text-red-400 border border-red-500/30`), date (`text-xs text-dimmed`), title (`text-sm font-bold text-primary truncate`), champion icon 20px if type is champion/matchup, first 80 chars of `what_happened` as preview (`text-muted text-xs`), tags as chips (`rounded-full px-3 py-1 text-xs font-normal bg-overlay text-muted`)
- Card content (expanded): all three sections with headings ("What happened", "What I learned", "Next time I will..."), edit button linking to `/personal-learnings/new?edit={id}`, delete button (`text-red-400/50 hover:text-red-400 text-xs transition-colors`)

**Champion icon URL:** `format!("https://ddragon.leagueoflegends.com/cdn/15.6.1/img/champion/{name}.png")` — inline helper function.

**Delete handler:** `spawn_local` calling `delete_learning(id)`, on success call `learnings.refetch()` and `toast.show.run((ToastKind::Success, "Learning deleted.".into()))`.

**Empty state** (per UI-SPEC copywriting):
- No learnings at all: heading "No learnings yet", body "Write your first reflection after a game. Start from a match or create one here.", CTA link "Write your first learning" pointing to `/personal-learnings/new`
- No learnings matching filters: "No learnings match these filters." with a reset link

### Form Page Component: `NewLearningPage`

Auth guard same as browse page.

**Query param pre-fill** (per D-12, D-13, UI-SPEC):
```rust
use leptos_router::hooks::use_query_map;
let query = use_query_map();
```
Read: `champion`, `opponent`, `match_id`, `result`, `event_ts`, `event_name`, `edit`, `tag_hint`.

**Edit mode detection:** If `query.read().get("edit")` is non-empty, call `get_learning(edit_id)` in a Resource and populate all signals from the returned record.

**State signals:**
- `learning_type: RwSignal<String>` — default "general", or "matchup" if both champion+opponent in URL, or "champion" if only champion
- `champion: RwSignal<String>` — from URL or empty
- `opponent: RwSignal<String>` — from URL or empty
- `what_happened: RwSignal<String>`
- `what_i_learned: RwSignal<String>`
- `next_time: RwSignal<String>`
- `selected_tags: RwSignal<Vec<String>>` — pre-select from `tag_hint` URL param if present
- `title: RwSignal<String>` — auto-generated but user-editable
- `title_edited: RwSignal<bool>` — tracks if user manually edited title
- `win_loss: RwSignal<String>` — from URL "result" param or empty
- `match_riot_id: RwSignal<String>` — from URL "match_id" param or empty
- `game_timestamp_ms: RwSignal<String>` — from URL "event_ts" param or empty
- `event_name_signal: RwSignal<String>` — from URL "event_name" param or empty
- `save_error: RwSignal<Option<String>>` — validation error message
- `is_editing: RwSignal<bool>` — true when editing existing entry

**Auto-title derivation** (per D-03, reactive — per pitfall 7):
Inside a `move ||` closure for `prop:value` on the title input:
```
if title_edited.get() { title.get() }
else match learning_type.get().as_str() {
    "matchup" => format!("{} vs {} — {date}", champion.get(), opponent.get()),
    "champion" => format!("{} — {date}", champion.get()),
    _ => format!("General — {date}"),
}
```
Where `{date}` is formatted as "Mar 27" (month abbreviation + day). Use `chrono::Utc::now().format("%b %d")` if chrono is available, otherwise hardcode using JS Date via `web_sys` under `#[cfg(feature = "hydrate")]` or just use a static approach.

**Event context banner** (per D-13, D-14, UI-SPEC):
When `event_name` and `game_timestamp_ms` are non-empty from URL:
```html
<div class="bg-elevated border border-divider rounded-lg p-3 text-sm text-secondary mb-4">
    "From match: {event_name} at {formatted_timestamp}"
</div>
```

**Type selector** (per D-01, UI-SPEC):
Three buttons: General | Champion | Matchup
- Active: `px-4 py-2 rounded-lg text-sm font-bold bg-accent text-accent-contrast`
- Inactive: `px-4 py-2 rounded-lg text-sm font-normal bg-elevated border border-divider text-secondary hover:text-primary hover:border-accent transition-colors`
- On click: set `learning_type` signal. Use `get_untracked()` pattern.

**Conditional fields** (per D-01, using `.into_any()` per CLAUDE.md rule 19):
- "matchup": show two `ChampionAutocomplete` (champion + opponent)
- "champion": show one `ChampionAutocomplete` (champion only)
- "general": show nothing

**Three text areas** (per D-02, UI-SPEC):
- Labels: "What happened", "What I learned", "Next time I will..."
- Style: `w-full bg-surface/50 border border-outline/50 rounded-lg px-3 py-2 text-primary text-sm placeholder-dimmed focus:outline-none focus:border-accent/50 transition-colors`
- `rows="4"`, use `prop:value` and `on:input`
- Inline validation: if empty on save attempt, show `<p class="text-red-400 text-xs mt-1">"This field is required to save."</p>` below

**Tag chips** (per D-06, D-07, UI-SPEC):
- Render all 8 `LEARNING_TAGS` as clickable chips
- Selected: `rounded-full px-3 py-1 text-xs font-bold bg-accent text-accent-contrast cursor-pointer`
- Unselected: `rounded-full px-3 py-1 text-xs font-normal bg-overlay text-muted hover:bg-overlay-strong hover:text-secondary transition-colors cursor-pointer`
- On click: toggle tag in `selected_tags` signal (add if absent, remove if present)

**Title input** (per D-03, UI-SPEC):
- `prop:value=move || { auto_title_or_manual }` using the reactive derivation above
- `on:input`: set `title.set(value)` and `title_edited.set(true)`
- Style: `w-full bg-surface/50 border border-outline/50 rounded-lg px-3 py-2 text-primary text-sm focus:outline-none focus:border-accent/50`

**Save handler** (per D-02, D-04):
- Validate all three sections non-empty, show inline errors if any are empty
- Serialize tags to JSON string: `serde_json::to_string(&selected_tags.get_untracked())`
- If editing: call `update_learning(...)`, on success toast "Learning saved." and navigate to `/personal-learnings`
- If creating: call `save_learning(...)`, on success toast "Learning saved." and navigate to `/personal-learnings`
- On error: set `save_error` signal with "Failed to save. Check your connection and try again."
- Use `spawn_local` (CLAUDE.md rule 24)
- Navigate via `window.location().set_href("/personal-learnings")` (hard nav for simplicity)

**Save button:** `bg-accent hover:bg-accent-hover text-accent-contrast font-bold px-5 py-2 rounded-lg text-sm transition-colors` — label "Save Learning"
**Cancel button:** `bg-elevated border border-divider text-secondary hover:text-primary px-5 py-2 rounded-lg text-sm transition-colors` — label "Discard Changes", navigates to `/personal-learnings`

**Page heading:** "New Learning" (or "Edit Learning" when `is_editing` is true) per UI-SPEC.

## Spacing Scale

Declared values (must be multiples of 4). Source: existing codebase patterns (nav.rs, stat_card.rs, ui.rs).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), tight inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `px-2 py-1`) |
| md | 16px | Default element spacing (`px-4`, `gap-4`) |
| lg | 24px | Section padding (`py-6`, `gap-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (`py-12` on empty states) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Nav bar height: 56px (`h-14`) — matches existing nav bar
- Mode toggle button: minimum 32px tall (`py-1.5`) for one-hand reachability
- Queue filter dropdown: minimum 36px tall (`py-2`) — dropdown affordance

## Typography

Source: existing codebase patterns extracted from stat_card.rs, nav.rs, ui.rs.

Two weights only: 400 (regular) and 600 (semibold). Hierarchy between Display and Heading is maintained by size alone (28px vs 20px).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 12px (`text-xs`) | 400 (regular) | 1.4 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Display | 28px (`text-3xl`) | 600 (semibold) | 1.1 |

Notes:
- LP display inside ranked badge uses Display (28px semibold) — the primary data point
- Tier + division label (e.g. "Gold II") uses Heading (20px semibold)
- All body text on match rows: Body (14px regular)
- Stat labels (e.g. "RANKED SOLO", "RECENT MATCHES"): Label with `uppercase tracking-wider` — matches StatCard pattern

## Color

Source: `input.css` semantic tokens. All values use semantic token names, not raw hex, to support dark/light theme switching.

| Role | Token | Dark value | Usage |
|------|-------|------------|-------|
| Dominant (60%) | `bg-base` | `oklch(0.13 0.004 264.665)` (gray-950) | Page background |
| Secondary (30%) | `bg-surface` / `bg-elevated` | gray-900 / gray-800 | Cards, sidebar, nav, ranked badge container, match rows |
| Accent (10%) | `bg-accent` / `text-accent` | yellow-400 (dark), yellow-500 (light) | See reserved list below |
| Destructive | `text-red-400` / `bg-red-500/10` | red-400 | Error banners, sync failure messages |
| Success | `text-emerald-400` / `bg-emerald-500/10` | emerald-400 | Sync success toasts, win indicators on match rows |

**Accent reserved for:**
1. Mode toggle active state — the active mode pill (Solo / Team) uses `bg-accent text-accent-contrast`
2. Primary CTA buttons — "Sync Matches" button on solo dashboard
3. LP gain number — positive LP delta (`+18 LP`) in match rows
4. Register/sign-in CTA (existing, unchanged)
5. App name logotype in nav (existing, unchanged)

**NOT accent:** Queue filter dropdown, nav links, ranked tier text, goal placeholder cards.

Additional semantic colors for this phase:
- Win row left border: `border-l-4 border-blue-500` (existing stats page convention to check)
- Loss row left border: `border-l-4 border-red-500/50`
- LP loss delta: `text-red-400` (e.g. "-15 LP")
- "Coming in a future update" placeholder label: `text-dimmed` — subdued to communicate not-yet-available state
- Team-only gate message banner: `bg-surface border border-outline` with `text-secondary` body — deliberately neutral (not red/error), it's an informational gate not a failure

## Component Inventory

New components this phase builds or extends. Each maps to a requirement.

### ModeToggle (new — nav bar)
**Requirement:** SOLO-01, D-01
- Location: Nav bar right side, left of ThemeToggle
- Anatomy: Two-pill toggle — `[Solo] [Team]` — where active pill is `bg-accent text-accent-contrast font-semibold`, inactive is `text-muted hover:text-secondary`
- Container: `bg-elevated rounded-lg p-0.5 flex` — pill group
- Size: `px-3 py-1 text-sm rounded-md` per pill
- State: Fetches current mode from server; optimistic update on click; persists to DB via server fn
- SSR fallback: render both pills, no active state until hydration resolves

### RegionDropdown (new — profile page)
**Requirement:** SOLO-05, D-06
- Location: Profile page Riot account linking section, inline with game_name/tag_line inputs
- Anatomy: `<select>` styled as `bg-surface/50 border border-outline/50 rounded-lg px-3 py-2 text-sm text-primary`
- Options: All 17 Riot API regions (EUW, NA, KR, EUNE, OCE, BR, LAN, LAS, TR, RU, JP, PH, SG, TH, TW, VN) plus a default "Select region" placeholder
- Validation: Required before save; shows inline error `text-red-400 text-xs` below select if missing on submit

### QueueFilter (new — stats page + solo dashboard)
**Requirement:** SOLO-03, D-12, D-13
- Anatomy: `<select>` with three options: "All Queues", "Solo/Duo", "Flex"
- Style: `bg-surface border border-outline/50 rounded-lg px-3 py-2 text-sm text-secondary` — compact, consistent with RegionDropdown
- Placement: Top-right of match history section header, same row as section title
- State: `RwSignal<Option<i32>>` — `None` = all, `Some(420)` = solo/duo, `Some(440)` = flex

### SoloDashboard page (new — `/solo-dashboard` route)
**Requirement:** SOLO-04, D-07
- Layout: Single column, `max-w-2xl mx-auto`, stacked sections with `gap-8` between
- Section order: 1. Ranked Badge, 2. Recent Matches, 3. Goal Placeholders

### RankedBadge (new — solo dashboard section)
**Requirement:** RANK-01, D-08, D-09
- Container: `bg-elevated border border-divider rounded-xl p-6` — card style matching StatCard
- Anatomy:
  - Left: Tier emblem image 64×64px (`w-16 h-16`) — sourced from Data Dragon CDN ranked emblems
  - Right column: tier + division (28px semibold, `text-primary`), LP (20px semibold, `text-secondary`), "Solo/Duo" label (12px, `text-dimmed uppercase tracking-wider`)
- Unranked state: Replace emblem with a greyed placeholder icon (SVG shield outline), text reads "Unranked"
- Loading skeleton: `SkeletonCard height="h-28"` — matches existing pattern

### RankedSnapshot sync indicator
**Requirement:** RANK-03, D-10
- "Sync Matches" button: `bg-accent hover:bg-accent-hover text-accent-contrast font-semibold rounded-lg px-4 py-2 text-sm` — primary CTA style
- Auto-sync indicator: no visible spinner; on page load sync happens silently if stale; on manual sync, button text changes to "Syncing..." and is `disabled` with `opacity-60 cursor-not-allowed`
- After sync: Show success toast via `ToastContext` ("Synced N matches") — 4s auto-dismiss

### GoalPlaceholderCard (new — solo dashboard section)
**Requirement:** SOLO-04, D-11
- Count: 3 cards in a `grid grid-cols-1 sm:grid-cols-3 gap-3`
- Each card: `bg-elevated border border-divider/50 rounded-xl p-4 opacity-60`
- Content per card: goal icon (SVG, `text-dimmed`), goal name (`text-secondary text-sm font-medium`), "Coming in a future update" label (`text-dimmed text-xs`)
- Goal names: "Rank Target", "CS per Minute", "Deaths per Game"
- No hover state — `cursor-default`, not interactive
- Note: D-11 specifies label text "Coming in Phase 15" — UI-SPEC intentionally uses "Coming in a future update" instead. Phase numbers are an internal planning detail; user-facing copy should not expose them.

### TeamModeGate (new — team-only pages in solo mode)
**Requirement:** D-02
- Not an ErrorBanner (red) — a neutral informational gate
- Container: `bg-surface border border-outline rounded-xl p-6 text-center`
- Body: `text-secondary text-sm` message + `<a>` CTA button with accent style

### Match row LP delta (enhancement — existing match rows)
**Requirement:** SOLO-04 specifics
- Added to match row right side: `+18 LP` in `text-accent font-semibold text-sm` or `-15 LP` in `text-red-400 font-semibold text-sm`
- Hidden when no ranked snapshot delta is available

## Implementation Notes for Executor

1. **Semantic tokens only** — never use hardcoded `oklch(...)` or `#hex` values in component markup. Always use `bg-base`, `bg-surface`, `bg-elevated`, `text-primary`, `text-secondary`, `text-muted`, `text-dimmed`, `border-divider`, `border-outline`, `bg-accent`, `text-accent-contrast` as defined in `input.css`. Exception: colored action buttons (e.g. win/loss indicators) may use Tailwind color scale directly (e.g. `border-blue-500`, `text-red-400`).

2. **ModeToggle SSR safety** — ModeToggle must render on SSR without knowing the active mode (no flash). Render both pills without active state in SSR; apply active state only after hydration resolves the user resource. Pattern matches how auth-gated nav links work in `nav.rs`.

3. **RankedBadge emblem** — Fetch tier emblem from Data Dragon CDN pattern: `https://ddragon.leagueoflegends.com/cdn/img/ranked-emblems/{tier}.png` where `{tier}` is uppercase (e.g. `GOLD`, `PLATINUM`). Use `<img>` with `w-16 h-16 object-contain` and a greyed fallback SVG shield for Unranked/loading.

4. **LP delta sign** — Always show sign: `+18 LP` (accent) or `-15 LP` (red-400). If delta is 0, do not show delta. If no ranked snapshot available, hide delta entirely.

5. **TeamModeGate placement** — Gate banner replaces the page body content, not the nav or page shell. The page title still renders above the gate. Pattern: early-return in the Suspense block when `mode == solo`.

6. **Queue filter persistence** — `RwSignal<Option<i32>>` only — do NOT persist queue filter to DB. It's a session-local UI preference. Reset to "All Queues" on page load.

7. **Auto-sync staleness** — Threshold is 10 minutes (600 seconds). Compare `last_sync_at` from user record against server timestamp. If stale or null, trigger sync server fn automatically on solo dashboard mount. Use same `spawn_local` + `Resource::refetch()` pattern as existing debounced save.

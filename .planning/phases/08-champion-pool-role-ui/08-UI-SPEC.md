# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: Champion Pool & Role UI.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS v4 with custom semantic tokens) |
| Preset | not applicable |
| Component library | none (Leptos 0.8 native components) |
| Icon library | Community Dragon SVG URLs (role icons); Data Dragon CDN (champion portraits) |
| Font | system font stack (no explicit font override in input.css) |

**Source:** `input.css` @theme block + existing codebase patterns. No shadcn gate applies — this is a Rust/Leptos project, not React/Next.js/Vite.

## Spacing Scale

Declared values (multiples of 4 only). All new UI must use these Tailwind utility classes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, badge inner padding, card close button position, card metadata area padding |
| sm | 8px (`gap-2`, `p-2`) | Card grid gap, tier bucket inner padding, role popover button gaps |
| md | 16px (`p-4`, `gap-4`) | Tier bucket padding, section separation inside card metadata area |
| lg | 24px (`p-6`, `mb-6`) | Tier section vertical spacing, detail panel padding |
| xl | 32px (`gap-8`) | Column layout gap between pool grid and detail panel |
| 2xl | 48px | Major section breaks on champion pool page (not used inline) |

Exceptions:
- Matchup opponent icon in notes list: 24px fixed width (`w-6 h-6`) — not on spacing scale, used as icon size
- Role badge on draft slot: `bottom-0 right-0` (flush corner, 0px) — badge sits at corner with no inset

## Typography

Established by the existing codebase. New UI in this phase must match these exact classes.

Two weights only: 400 (regular) for body and secondary text; 700 (bold) for all labels, headings, and micro badges.

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 12px | 400 (regular) | 1.4 | `text-xs` |
| Heading | 30px | 700 (bold) | 1.2 | `text-3xl font-bold` |
| Micro | 10-11px | 700 (bold) | 1.0 | `text-[10px] font-bold` (tier badges, meta tags) |

**Source:** Observed from `champion_pool.rs` page heading (`text-3xl font-bold`), pill labels (`text-sm`), tier headings (`text-xs font-semibold uppercase tracking-wider` — pre-Phase-8 pattern, normalized to `text-xs font-bold` in all Phase 8 additions), and meta badges (`text-[9px] font-bold`).

Phase 8 additions:
- Champion card name: `text-xs font-bold` (12px/700) — truncated single line below portrait
- Card comfort stars: `text-xs` (12px/400) — secondary metadata
- Card stats line: `text-[10px]` (10px/400) in `text-dimmed` — tertiary, de-emphasized
- Role popover labels: `text-xs` (12px/400) — 5 role buttons, centered under icon

## Color

Uses semantic tokens from `input.css`. Do not use hardcoded hex values.

| Role | Semantic Token | Dark Value | Light Value | Usage |
|------|---------------|------------|-------------|-------|
| Dominant (60%) | `bg-base` | gray-950 | gray-50 | Page background |
| Secondary (30%) | `bg-surface` / `bg-elevated` | gray-900 / gray-800 | white / gray-100 | Tier bucket backgrounds, champion cards, detail panel |
| Accent (10%) | `bg-accent` / `border-accent` / `text-accent` | yellow-400 | yellow-500 | See reserved list below |
| Destructive | `bg-red-700 text-white` (literal) | red-700 | red-700 | Remove champion from pool (× button on card) |

**Accent reserved for:**
1. Selected champion card border + box-shadow glow (D-03)
2. Tier bucket drag-over highlight border (`border-accent/60`) and background tint (`bg-accent/10`) (D-06)
3. "Add Champion" primary CTA button (`bg-accent text-accent-contrast`)
4. Active tab underline in champion detail panel
5. User-confirmed role badge border (solid, full opacity) (D-14)

**Semi-transparent variants in use:**
- `border-accent/60` — selected card border, drag-over tier border
- `bg-accent/10` — drag-over tier bucket background tint
- `hover:border-accent/30` — unselected card hover state

**Role badge opacity states (D-14):**
- Auto-guessed: badge rendered at `opacity-50`, `border-dashed border-outline`
- User-confirmed: badge rendered at `opacity-100`, `border-solid border-accent`

## Component Inventory

### Champion Pool Card (new — replaces pill entries)

**Dimensions:** ~140px wide, determined by 3-column grid within tier bucket. Portrait is `w-full aspect-square object-cover object-top`.

**Structure:**
```
[card: relative bg-elevated border rounded-xl overflow-hidden cursor-pointer group]
  [portrait: img w-full aspect-square object-cover object-top]
  [× button: absolute top-1 right-1, opacity-0 group-hover:opacity-100]
  [metadata: p-1 flex flex-col gap-1]
    [name: text-xs font-bold text-primary truncate]
    [comfort stars: text-xs text-muted]
    [meta tag badge: text-[9px] px-1 rounded border font-bold — reuse existing cls logic]
    [stats line: text-[10px] text-dimmed — optional, shown only if stats exist]
```

**States:**
- Default: `border-divider`
- Hover: `hover:border-accent/30`
- Selected: `border-accent/60` + `box-shadow: 0 0 8px color-mix(in oklch, var(--color-accent) 30%, transparent)`
- Dragging (source): `opacity-50` applied via JS `on:dragstart` → `classList.add("opacity-50")`

**Source:** D-01, D-02, D-03, D-04; CONTEXT.md decisions; RESEARCH.md Pattern 1.

### Tier Bucket Drop Zone (modified existing)

**Modification:** Add `on:dragover`, `on:dragleave`, `on:drop` to existing tier `<div>`. Dragover state tracked via single `RwSignal<Option<&'static str>>` at page level.

**States:**
- Default: `border rounded-xl p-4 {tier_color(tier)}` (existing)
- Drag-over active: append `border-accent/60 bg-accent/10` (D-06)

**Source:** D-05, D-06; RESEARCH.md Pattern 2.

### Role Badge Overlay (new — on draft pick slots)

**Size:** 16px icon (`w-4 h-4`) inside a 20px container (`w-5 h-5`).
**Position:** `absolute bottom-0 right-0` on the pick slot container (which must have `relative`). Flush corner, no inset.
**Background:** `bg-base/80` rounded circle (`rounded-full`) for contrast against champion portrait.

**Accessibility:** Badge button must have both `title="{tooltip text}"` and `aria-label="{tooltip text}"` attributes for screen-reader robustness. Tooltip text follows the Copywriting Contract rules for role auto-guess and role confirmed states.

**States:**
- Auto-guessed: `opacity-50 border border-dashed border-outline`
- User-confirmed: `opacity-100 border border-solid border-accent`
- No role: show dimmed question-mark placeholder using existing `text-dimmed`

**Source:** D-11, D-14; CONTEXT.md.

### Role Selection Popover (new — triggered by role badge click)

**Dimensions:** Fixed `w-48` (192px) container, 5 buttons in a single row or 2-3 column grid.
**Position:** Absolute, positioned above the pick slot. Use `bottom-full mb-1` relative to badge.
**Background:** `bg-surface border border-divider rounded-xl p-2 shadow-lg`
**Dismiss:** Click outside triggers close (via `on:blur` on the popover or global click handler with `ev.stop_propagation()` on popover content).

**Button structure (per role):**
```
[button: flex flex-col items-center gap-1 p-1 rounded-lg hover:bg-overlay cursor-pointer]
  [img: w-6 h-6 (Community Dragon SVG URL)]
  [span: text-[10px] text-muted capitalize]
```

**Active role:** button gets `bg-overlay text-primary` to indicate current selection.

**Source:** D-12; CONTEXT.md.

### Matchup Note Form (modified — matchup type only)

**When note type = "matchup":**
- Replace free-text title `<input>` with `<ChampionAutocomplete>` component
- Title field auto-populates as "vs {champion name}" when autocomplete selection fires
- ChampionAutocomplete uses existing `on_select: Callback<String>` prop pattern

**Matchup note display in list (D-10):**
```
[row: flex items-center gap-2]
  [opponent icon: img w-6 h-6 rounded-full] ← Community Dragon champion icon
  ["Matchup" label: text-xs text-muted]
  [vs champion name: text-xs text-secondary]
```

**Source:** D-08, D-09, D-10; CONTEXT.md.

## Interaction Contracts

### Drag-and-Drop (Champion Pool Tier Changes)

| Event | Behavior |
|-------|----------|
| `dragstart` on card | Set `DataTransfer` keys: `text/x-champion` (champion id), `text/x-role` (role), `text/x-source-tier` (current tier). Apply `opacity-50` to source card via DOM class. |
| `dragover` on tier bucket | `ev.prevent_default()`. Set `dragging_over_tier` signal to this tier key. |
| `dragleave` on tier bucket | Set `dragging_over_tier` to `None`. |
| `drop` on tier bucket | `ev.prevent_default()`. Set `dragging_over_tier` to `None`. If source tier != target tier, `spawn_local` → `set_champion_tier(champ_id, role, target_tier)` → `pool.refetch()`. If same tier, no-op. |
| `dragend` on card | Remove `opacity-50` from source card (cleanup regardless of drop success). |

No within-tier reordering. Drop on same tier is a no-op (D-07).

### Role Badge Popover (Draft Pick Slots)

| Event | Behavior |
|-------|----------|
| Click on role badge | Toggle popover open/closed. `ev.stop_propagation()`. |
| Click on role button in popover | Set role for this slot. Close popover. Update `role_assignments` signal (per-slot `RwSignal<Option<String>>`). |
| Click outside popover | Close popover. Implemented via global click listener or `on:focusout` on popover wrapper. |
| Champion placed in slot | Auto-guess role from `Champion.tags` heuristic. Set badge to auto-guessed state (50% opacity, dashed). |
| Slot cleared | Clear role assignment for that slot. |

Role state is stored in a `Vec<RwSignal<Option<String>>>` of length 10 (one per pick slot), initialized in the `DraftBoard` or `DraftPage` parent.

### Champion Card Selection (Champion Pool)

| Event | Behavior |
|-------|----------|
| Click on card | Set `selected_champion` signal. Show detail panel. `ev.stop_propagation()` from × button must prevent card selection on delete. |
| Click on × button | `ev.stop_propagation()`. `spawn_local` → remove champion → `pool.refetch()`. If removed champion was selected, clear `selected_champion`. |

## Champion Class → Role Heuristics (D-13)

Used for auto-guess on drag-drop into draft slot. Best-effort, not authoritative.

| Champion.tags contains | Guessed role |
|------------------------|--------------|
| "Marksman" | bot |
| "Support" | sup |
| "Assassin" (without "Fighter") | mid |
| "Mage" (without "Support") | mid |
| "Tank" or "Fighter" (without "Assassin") | top |
| "Fighter" + any jungle keyword | jungle |
| Ambiguous / multi-tag | top (fallback) |

Jungle detection: maintain a static list of well-known junglers (`["Hecarim", "Vi", "Amumu", "Warwick", "Xin Zhao", "Rammus", "Nunu & Willump", "Shyvana", "Volibear"]`) as override before tag-based logic.

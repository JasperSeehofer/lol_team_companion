### 60/30/10 role mapping for extended tokens

The vault's 60/30/10 split applies to the extended demacia/pandemonium tokens as follows:

- **60% (primary surfaces — `bg-base`, `bg-surface`):** `--t-base`, `--t-surface`. The dominant canvas. Demacia: deep navy. Pandemonium: near-black.
- **30% (secondary surfaces — cards, sidebar, elevated panels):** `--t-elevated`, `--gold-1` (Demacia gilt card backgrounds), `--lapis-1` (Demacia banner fields), `--ivory` (Demacia journal backgrounds). These appear on component surfaces, not on page backgrounds.
- **10% (accent — sparingly, reserved for primary CTAs, on-deck draft halo, active nav state):** `--t-accent` (`gold-2` in Demacia, electric pink in Pandemonium). `--gold-3` and `--lapis-2` are ornamental sub-tones within this tier (engraved recesses, enamel), not interactive accents. `--accent-2` and `--accent-3` (Pandemonium) are decorative-only and do not count toward interactive accent budget.

### Theme application to `<html>`

`[data-theme="demacia"]` / `[data-theme="pandemonium"]` on `<html>` element (not `<body>`), matching Leptos SSR rendering constraints. Semantic tokens defined under `:root` fall through for both themes; theme blocks override only what changes.

### `.canvas-grain` utility class

Port the procedural grain + vignette from `themes.css` as a CSS utility class `canvas-grain` in `input.css`. Used on full-page wrapper divs for painterly backgrounds. Has `::before` and `::after` pseudo-elements (radial gradients + SVG fractal-noise filter). Do not use `@apply` for this — plain CSS in `input.css` `@layer utilities` or directly as a selector.

### Theme persistence

Theme stored on the DB `user` record (field: `theme`, default `'demacia'`). Read on page load via `get_current_user`; applied as `data-theme` attribute on `<html>` server-side where possible, and hydrated client-side. `theme_toggle.rs` becomes a 2-option toggle (Demacia | Pandemonium), replacing the current 5-accent picker.

|-----|------|-----------|----------------------------------|

| Profile | user | Captain's Folio | `/profile`, `/team/dashboard`, `/team/roster`, `/team-builder` |

**Secondary nav group** (separator then smaller links): Design shows `Draft`, `Tree`, `Pool` as secondary — these become prominent sub-nav within the Strategy hub in the Leptos implementation.

### Nav anatomy (per `app.jsx` TopNav)

```
[CompanionSigil]  [Strategy] [Live] [History] [Profile] | [Draft] [Tree] [Pool]    [match-day clock]  [ThemeToggle]
```

- Container: `position: sticky; top: 0; z-index: 50` with `backdrop-filter: blur(14px) saturate(1.1)`
- Demacia: `border-bottom: 1px solid var(--t-divider)` (gold-tinted line)
- Pandemonium: `border-bottom: 1px solid var(--t-accent)` (electric pink line)
- Padding: `16px 64px` (Demacia), `16px 48px` (Pandemonium)
- Grid: `grid-template-columns: auto 1fr auto auto; gap: 28px`

### CompanionSigil (logo mark)

- Demacia: shield SVG (`viewBox="0 0 22 26"`, filled `var(--accent)`) + "COMPANION" in `font-imperial` uppercase, `letter-spacing: 0.18em`
- Pandemonium: "COMPANION_" in `font-glitch`, `letter-spacing: 0.16em`, `color: var(--accent)`

### NavBtn active/inactive states

- Active: `bg-accent-soft border border-outline text-accent` — solid accent tint background
- Inactive: `text-muted hover:text-secondary` — no background
- Secondary (draft/tree/pool): `text-dimmed hover:text-muted` — further reduced
- Padding: `8px 12px` (Demacia), `8px 16px` (Pandemonium)
- Rounded: `rounded-md`
- No `outline:none` — use `focus-visible:ring-2 focus-visible:ring-accent/50`

### ThemeToggle

Replaces current 5-color picker. Two states: Demacia / Pandemonium.

- UI: pill toggle or two labeled buttons; icon per theme (shield for Demacia, flame/spark for Pandemonium)
- On change: update `user.theme` in DB via server fn, update `data-theme` on `<html>` immediately client-side

### URL paths under hubs

Gsd-planner decides whether to nest (`/strategy/draft`) or keep flat (`/draft`). This spec preserves flat paths to avoid breaking any existing bookmarks; plan-phase overrides if it has a strong reason to restructure.

## Draft Board Layout

Source: `draft-boards.jsx` — three design variants. Implementation uses the **War Table (Demacia)** as the primary variant.

### Overall layout

```
┌────────────────────────────────────────────────────────────┐
│  DraftHeader: [logo+phase badge]  [opp]  [timer][Save][→Plan]│
├────────────────────────────────────────────────────────────┤
│  Hero strip: [Our team name ←]  [vs. ornament]  [→ Opponent]│
│  HeraldicDivider                                            │
├──────────────────────┬─────────────────────────────────────┤
│  [Ban strip — blue 3]│  [Ban strip — red 3]                 │
│  (wax-seal style)    │  (wax-seal style)                    │
├──────────────────────┼─────────────────────────────────────┤
│  BLUE SIDE PICKS     │  RED SIDE PICKS                      │
│  [5 pick slots]      │  [5 pick slots]                      │
│  Each: 64px champ    │  Each: role icon + name + player     │
│  On-deck: gold halo  │  Empty: outlined placeholder         │
├──────────────────────┴─────────────────────────────────────┤
│  Champion picker (full-width grid, searchable)              │
└────────────────────────────────────────────────────────────┘
```

### DraftHeader

- Height: ~56px
- Contents: logo sigil, phase badge (accent), opponent name in `font-display italic`, timer in `font-mono border border-outline rounded-md px-2.5`, "Save draft" outline btn, "To plan" primary btn
- Background: `bg-surface/80 backdrop-blur`
- Border-bottom: `border-divider`

### Ban slots

- Size: 64×64px champion tile, `border-radius: 50%` (circular)
- Border: `2px solid var(--gold-3)` → maps to a demacia-specific border class
- Banned state: grayscale + brightness-50 + diagonal red line overlay (`bg-danger h-px w-full absolute`)
- Empty ban slot: circular `bg-elevated border-2 border-divider/50`
- Label below: `text-xs text-dimmed uppercase tracking-wider` — "Forsworn" (Demacia), or "BAN" (fallback)

### Pick slots

- Size: 64×64px ChampTile (square, `rounded-md`)
- Layout: `grid grid-cols-[64px_20px_1fr] gap-3 items-center`
- Role icon: 20×20px SVG mask (Community Dragon URLs — self-host or use CDN; see note)
- Player name: `text-sm font-semibold text-primary font-display italic`
- On-deck (current turn): gold halo — `ring-2 ring-accent ring-offset-2 ring-offset-base` + subtle glow `shadow-[0_0_14px_var(--color-accent)]`
- Empty slot: `bg-elevated border border-dashed border-outline/50 rounded-md` with faint "?" or role icon at 50% opacity
- Locked/filled: full opacity, `border border-outline/30`

### Slot interaction

Existing `draft_board.rs` patterns preserved:
- **First click on filled slot:** highlights it (ring state)
- **× badge click on highlighted slot:** clears it (`ev.stop_propagation()` prevents outer click re-firing)
- **Click on empty slot:** opens champion picker if it is the active pick/ban phase

### Champion picker (within draft page)

Full-width grid below the draft board. Same component as `src/components/champion_picker.rs`, restyled:
- Grid: `grid grid-cols-[repeat(auto-fill,minmax(56px,1fr))] gap-2`
- Each tile: 56×56px `ChampTile`, hover: `ring-1 ring-accent/40 cursor-pointer`
- Search bar above grid: full-width `input` — `bg-surface/50 border border-outline/50 rounded-lg px-4 py-3 text-primary text-sm` + search icon left
- Filtered state: tiles not matching search are `opacity-30 pointer-events-none`
- Side toggle (Blue/Red): pill buttons top-right of picker

### Phase indicator

Phase labels use `font-imperial uppercase letter-spacing: 0.18em`. Current phase shown in accent-colored badge. Timer shown as `MM:SS` in `font-mono`.

## Tree Graph Interactions

Source: `tree-drafter.jsx` and existing `src/components/tree_graph.rs` (709 lines).

### Node states (5 total)

| State | Visual treatment | `--color-*` usage |
|-------|-----------------|-------------------|
| `locked` | Filled champ tile, solid gilt border, 100% opacity | `border-outline` |
| `selected` | Gold halo + animated dash stroke on connecting edge | `ring-accent shadow-accent/30` |
| `alternate` | Visible, 78% opacity, lapis/oxblood border tint | `text-secondary border-divider` |
| `ghost` | 32% opacity, pencil-sketch feel (dashed border) | `opacity-30 border-dashed border-divider/50` |
| `leaf` | Terminal node, glowing aureole, outcome % badge | `ring-accent/60 shadow-accent/20` |

### Node anatomy

```
┌──────────────────────────┐  w: 144px, h: 56px
│  [ChampTile 44px]  B1    │  phase label: text-xs font-mono text-muted
│  Champion Name           │  text-sm font-semibold text-primary
│  [win% badge]  [side dot]│  win%: text-xs; side dot: us=info, them=danger
└──────────────────────────┘
```

- Root node: wider, centered, `bg-elevated border border-outline` — no champ tile
- `kind=pick`: square ChampTile 44px inside
- `kind=ban`: ChampTile 44px with diagonal ban overlay
- `kind=leaf`: circular, larger (64px), shows `outcome%` in big text + label below

### Edge rendering

SVG paths (cubic bezier) connecting nodes:
- Selected path: `stroke-accent stroke-width-2` + `stroke-dasharray` animated via `@keyframes` (dashes flow forward along the path)
- Alternate path: `stroke-secondary stroke-width-1.5 opacity-60`
- Ghost path: `stroke-muted stroke-width-1 opacity-25 stroke-dasharray="4 4"`

Side color for path tint:
- `side=us`: `stroke-info` (lapis blue — `var(--lapis-1)` / `var(--info)`)
- `side=them`: `stroke-danger`

### Tree interactions

- **Click node:** selects it as the active node; its subtree becomes `selected`; sibling subtrees become `alternate`; dead branches become `ghost`
- **Click empty canvas:** deselects (all visible nodes revert to `alternate` or node's default state)
- **Node label inline edit:** double-click on node label → `contenteditable` or `<input>` overlay, auto-saves after 2s debounce (existing pattern from Phase 12, `debounced auto-save` rule in MEMORY.md)
- **Add child node:** "+" button appears on hover of any non-ghost node; opens champion picker modal
- **Delete node:** hover reveals "×" badge on non-root nodes; click shows confirmation inline (not a modal): "Remove this branch and all descendants?" [Remove] [Cancel]
- **Zoom/pan:** SVG viewBox manipulation via mouse drag and wheel scroll; reset button in corner

### ChampionAutocomplete (within tree nodes)

Used when adding a champion to a node. `src/components/champion_autocomplete.rs` restyled to match design's node picker:
- Input: `bg-elevated border border-outline rounded-md px-3 py-2 text-sm text-primary`
- Dropdown: `bg-surface border border-divider rounded-md shadow-lg` max-height `240px` overflow-scroll
- Each item: `flex items-center gap-2 px-3 py-2 hover:bg-elevated cursor-pointer` with 24px ChampTile thumbnail + name
- Keyboard: `↑↓` navigate, `Enter` selects, `Esc` dismisses

## Implementation Notes for Executor

1. **Read the handoff README first.** `/tmp/lol-design-handoff/lol-team-companion-app/README.md` and `index.html` establish the load order for the prototype. Walk `data.jsx` → `components.jsx` → `foundations.jsx` → `draft-boards.jsx` etc. before implementing any page.

2. **Semantic tokens bridge.** The design uses `var(--text-strong)`, `var(--surface)`, `var(--line-soft)` etc. These map to the `--t-*` aliases in `input.css`. Before implementing: add the demacia/pandemonium token blocks that override these `--t-*` values under `[data-theme="demacia"]` and `[data-theme="pandemonium"]` selectors. Components continue using `bg-base`, `text-primary` etc. — they automatically inherit the new theme values.

3. **SVG color tokens in Tailwind v4.** For SVG `fill` and `stroke`, use `class="fill-current"` with parent `text-accent` where Tailwind utilities work. For cases where CSS utilities don't reach SVG attributes (e.g. inline `<svg>` in a Leptos view macro), use `style="stroke: var(--color-accent)"`. Mirror the pattern already used in `tree_graph.rs`.

4. **Per-page review gate.** After implementing each page: run dev server → `npx agent-browser screenshot http://127.0.0.1:3020/{route}` → compare with design file → user approves → atomic commit. Do not batch multiple pages into one commit without approval.

5. **Font loading order.** `@font-face` declarations must appear in `input.css` before `@import "tailwindcss"` or at minimum before the `@theme` block, so Tailwind processes them. Test by checking the Network tab for font requests — all should be from `127.0.0.1`, not `fonts.googleapis.com`.

6. **Leptos SSR + `data-theme` attribute.** Set `data-theme` on `<html>` in the server-rendered shell (`src/app.rs` outer shell or `src/main.rs`), reading from the session user's `theme` field. This avoids a flash of un-themed content. Fall back to `"demacia"` when no user is authenticated.

7. **`theme_toggle.rs` cleanup.** Delete the `[data-accent="*"]` CSS blocks from `input.css` after confirming no components reference them. Replace toggle UI with a two-state Demacia/Pandemonium toggle. Update server fn to write `theme` field on `user` table.

8. **G-12 sweep.** Before each page commit: search for `outline: none` or `outline:none` in the file. Every instance must have a companion `focus-visible:ring-*` class or `:focus-visible { outline: ... }` CSS. Grep: `grep -n "outline.*none" src/pages/{page}.rs`.

9. **G-01 sweep.** Before the final commit on `input.css`: `grep -n "fonts.googleapis\|fonts.gstatic" input.css` must return zero matches.

10. **Hidden design asset.** The bundle contains `uploads/draw-92acceeb-9fd2-499d-84e4-12ff75b7ab5d.png`. Inspect it before implementing the closed-beta landing; it may be the reference background image or a composition sketch that informs the FLUX prompt.

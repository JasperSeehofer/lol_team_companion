## /gsd-ui-review Acceptance (6-Pillar Audit)

The phase's authoritative validation gate is `/gsd-ui-review 17`. Required PASS verdicts on:

1. **Visual coherence** — semantic tokens consistent, typography hierarchy enforced, no raw hex
2. **Accessibility** — focus states (G-12 ring replacement), color contrast meets WCAG AA, keyboard navigation, ARIA on interactive elements
3. **Responsiveness** — desktop-first, but no broken layouts at 1920×1080 / 1440×900 / 1280×720
4. **Information density** — no empty states without messaging, skeleton loaders on every Resource page
5. **Microinteractions** — toast feedback on mutations, hover states, focus rings
6. **Performance** — no Google Fonts CDN (G-01), local font subsets, image lazy-load, reasonable bundle size

If ANY pillar fails, the phase is not complete. `/gsd-ui-review` produces a structured report; gaps feed back into a follow-up plan or `/gsd-plan-phase 17 --gaps`.

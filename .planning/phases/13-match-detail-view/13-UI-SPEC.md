## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, item slot gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell padding, filter button gap |
| md | 16px | Default element spacing, card inner padding, section header margin |
| lg | 24px | Section padding, scoreboard table outer padding |
| xl | 32px | Layout gaps between major sections (scoreboard → timeline → performance) |
| 2xl | 48px | Major section breaks (page top padding) |
| 3xl | 64px | Page-level spacing (not used in dense data views) |

Exceptions:
- Item icon slots: 24px × 24px squares with 4px gap between them (6 item slots in a row) — uses `gap-1`
- Champion icon in scoreboard: 28px × 28px
- Timeline event markers: 8px × 8px circles for minor events (tower kills, champion kills); 12px × 12px for mid-significance events (dragon kills, inhibitors); 16px × 16px for major objective events (baron, herald). The three-tier size family (8/12/16px) encodes event significance levels — all multiples of 4, with a clear 4px step between each tier.
- Touch targets for filter toggle buttons: minimum 32px height (desktop-first, no 44px mobile requirement)
- User's own scoreboard row highlight: 4px left border accent stripe (not a spacing value, a visual treatment)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 12px (text-xs) | 400 (normal) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-3xl) | 600 (semibold) | 1.2 |

Source: pattern extracted from `solo_dashboard.rs` and `opponents.rs`. Section headings (`h2`) consistently use `text-xl font-semibold`. Page titles use `text-3xl font-semibold`. Table body data uses `text-sm`. Metadata labels use `text-xs`. Size contrast (12px / 14px / 20px / 28px) plus uppercase tracking on column headers provides sufficient hierarchy with only two weight steps.

Applied to this phase:
- Page title "Match Detail": text-3xl font-semibold text-primary
- Section headings ("Blue Team", "Red Team", "Timeline", "My Performance"): text-xl font-semibold text-primary
- Scoreboard data rows: text-sm font-normal for all columns including summoner name
- KDA numerals: text-sm font-semibold text-primary
- Column headers: text-xs font-normal text-muted uppercase tracking-wider
- Timeline tooltip text: text-sm text-secondary
- Performance bar labels: text-sm text-secondary
- Performance bar numbers: text-sm font-semibold text-primary
- Performance comparison text verdict (e.g. "Above average"): text-xs text-muted — included alongside bar chart numbers

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | bg-base (oklch 0.13 dark / oklch 0.985 light) | Page background |
| Secondary (30%) | bg-surface / bg-elevated | Scoreboard table background, section cards, timeline bar track |
| Accent (10%) | bg-accent / text-accent (yellow-400 default) | User's own row highlight left border; active filter toggle button background; "Add Learning" CTA button |
| Destructive | red-500/10 bg + red-400 text | Error banner on API failure (existing ErrorBanner component) |

Accent reserved for:
1. Left border stripe on user's own scoreboard row (4px solid border-accent)
2. Active state of timeline filter toggle buttons (bg-accent text-accent-contrast)
3. Primary "Add Learning" CTA button (bg-accent hover:bg-accent-hover text-accent-contrast)

Blue team color (semantic, non-accent): `bg-blue-500/20 border-blue-500/40 text-blue-300` — used for Blue team table header row background and timeline event markers belonging to Blue team.

Red team color (semantic, non-accent): `bg-red-500/20 border-red-500/40 text-red-300` — used for Red team table header row background and timeline event markers belonging to Red team.

Win/loss tinting on scoreboard rows: win rows get `bg-blue-500/5` tint; loss rows get `bg-red-500/5` tint. User's own row overrides with `border-l-4 border-accent bg-accent/10`.

Skeleton shimmer: `animate-pulse bg-elevated` — existing SkeletonCard/SkeletonLine pattern from `src/components/ui.rs`.

## Component Inventory

New components introduced in this phase:

### ScoreboardTable
Location: `src/pages/match_detail.rs` (inline, not a separate component file unless complexity warrants extraction)
Props: `participants: Vec<MatchParticipant>`, `user_puuid: String`, `side: "blue" | "red"`
Layout: full-width table, columns = Champion icon | Summoner name | KDA | Items (6 slots) | Damage | Gold | Vision Score
Row height: 48px to accommodate champion icon + item icons
Column widths: champion 36px | summoner 160px (flex-1) | KDA 80px | items 180px | damage 80px | gold 80px | vision 64px

### TimelineBar
Location: `src/pages/match_detail.rs`
Structure: horizontal bar (full width, height 40px) with `bg-elevated border border-divider rounded-lg` as track; event markers positioned absolutely by `(event_timestamp / game_duration_ms) * 100%`
Event marker shapes by type:
- Dragon kill: circle, 12px, team-colored
- Baron/Herald: circle, 16px, team-colored, bold outline
- Tower kill: square, 8px, team-colored
- Inhibitor: diamond (rotated square), 12px, team-colored
- Champion kill: circle, 8px, team-colored; first blood gets star icon overlay
- Teamfight (4+ within 10s window): larger circle, 16px, split blue/red gradient
- Ward placement (user's own): small dot, 8px, accent-colored
- User's own events: accent-colored ring/glow — `ring-2 ring-accent ring-offset-1 ring-offset-base`
Click behavior: clicking a marker sets `selected_event: RwSignal<Option<TimelineEvent>>` which renders a detail section below the bar
Hover behavior: tooltip via `title` attribute on the marker element (native browser tooltip, consistent with item icons per D-04)

### TimelineFilterBar
Location: inline above TimelineBar
Structure: row of toggle buttons, `flex gap-2 flex-wrap mb-3`
Button style inactive: `bg-elevated border border-divider text-muted text-xs px-3 py-1.5 rounded-full hover:border-outline hover:text-secondary transition-colors`
Button style active: `bg-accent text-accent-contrast text-xs px-3 py-1.5 rounded-full font-semibold`
Filter categories: "Objectives" | "Towers" | "Kills" | "Wards" | "Teamfights"
Default state: all filters active

### PerformanceBarChart
Location: `src/pages/match_detail.rs`
Structure: vertical stack of bar rows, each row = label (120px) + bar track (flex-1) + number (64px) + verdict (80px)
Bar track: `bg-elevated h-4 rounded-full relative`
Bar fill: `bg-accent/70 h-full rounded-full` sized via `style="width: {pct}%"`
Average marker: `absolute top-0 h-full w-0.5 bg-muted/50` positioned at `left: {avg_pct}%`
Verdict text: text-xs text-muted, values: "Above average" | "Average" | "Below average" (based on whether value exceeds avg by >10%)
Comparison toggle: pill toggle above the section — "vs Game Average" / "vs Lane Opponent" — same pattern as queue filter pills on solo dashboard

### ItemSlot
Location: inline in ScoreboardTable (not extracted unless reused)
Structure: `<img src="https://ddragon.leagueoflegends.com/cdn/{version}/img/item/{item_id}.png" class="w-6 h-6 rounded" title="{item_name}" />` or empty slot `<div class="w-6 h-6 rounded bg-elevated border border-divider/30" />` when no item
Six slots rendered in `flex gap-1`

## Interaction Contract

### Match row → detail navigation
- Stats page and solo dashboard match rows gain an `<a href="/match/{riot_match_id}">` wrapper
- Entire row is clickable — `cursor-pointer hover:bg-elevated/50 transition-colors`
- No separate "View" button — the row itself is the affordance

### Timeline markers
- Hover: native `title` tooltip (matches item icon pattern from D-04)
- Click: sets `selected_event` signal, renders detail panel below the bar with `bg-surface border border-divider rounded-lg p-4 mt-3`
- Clicking the same marker again: clears the selection (panel collapses)
- Clicking outside the bar: does NOT clear selection (selection is persistent until user clicks another marker or clicks the active marker again)

### Performance comparison toggle
- Pill toggle with two options — clicking the inactive option updates `comparison_mode: RwSignal<ComparisonMode>`
- When lane opponent data is unavailable (role detection failed), "vs Lane Opponent" is disabled: `opacity-40 cursor-not-allowed` with tooltip "Lane role data not available for this match"

### On-demand fetch flow
1. Page mounts, `fetch_match_detail(match_id)` server fn called
2. Server checks DB cache first — if hit, returns immediately
3. If miss, calls Riot API `get_match` + `get_timeline`, stores to DB, returns result
4. Suspense renders skeleton during the fetch; fills content on resolve
5. On error: ErrorBanner + "Retry load" button that re-invokes the resource `.refetch()`

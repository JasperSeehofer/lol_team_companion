## Spacing Scale

Standard 8-point scale. All values are multiples of 4. These match the Tailwind scale already in use across the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, badge padding |
| sm | 8px (`gap-2`, `p-2`) | Inline element spacing, toast button gaps |
| md | 16px (`gap-4`, `p-4`) | Default card padding, form field spacing |
| lg | 24px (`gap-6`, `p-6`) | Panel padding, section spacing |
| xl | 32px (`gap-8`) | Layout column gaps |
| 2xl | 48px (`py-12`) | Empty state vertical padding (existing `py-12` = 48px) |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Toast overlay: `top-4` (16px from viewport top), `z-50` fixed overlay — not a layout gap
- Toast min-width: `min-w-64` (256px) — functional constraint, not spacing token

## Typography

Derived from frequency analysis of existing page/component files. The codebase uses a 4-size scale with 3 weights.

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm` |
| Label / Caption | 12px | 500 (medium) | 1.4 | `text-xs font-medium` |
| Heading / Section | 18px | 600 (semibold) | 1.3 | `text-lg font-semibold` |
| Display / Page title | 30px | 700 (bold) | 1.2 | `text-3xl font-bold` |

Weight constraint: Only 2 weights are permitted in new components added this phase:
- **Regular (400)** — body copy, descriptions, messages
- **Semibold (600)** — labels, CTA button text, headings

Note: `font-bold` (700) exists on the logo/brand element and stat numbers — do not introduce `font-bold` on new Phase 5 components.

## Color

All values use semantic tokens from `input.css`. Do not use hardcoded hex or Tailwind palette classes except for semantic-override cases noted below.

| Role | Semantic Token | Dark Value | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `bg-base` | gray-950 | Page background |
| Secondary (30%) | `bg-surface` / `bg-elevated` | gray-900 / gray-800 | Cards, skeleton blocks, panel backgrounds |
| Accent (10%) | `bg-accent` / `text-accent` | yellow-400 (default) | CTA buttons, links, highlighted states |
| Destructive | `bg-red-500/10`, `border-red-500/30`, `text-red-400` | — | Error toasts, error banners only |
| Success semantic | `bg-emerald-500/20`, `border-emerald-500/40`, `text-emerald-300` | — | Success toasts only |

Accent reserved for:
- Empty state CTA button background (`bg-accent text-accent-contrast`)
- Active/hover states on navigation links
- Inline "View" link text in the post-game action item banner (`text-accent`)

Accent is NOT used for: skeleton backgrounds, toast containers, inline status text, error states.

### Semantic-override exceptions (permitted literal colors)

Per CLAUDE.md: colored buttons with white text keep `text-white` literally.
- Error toast dismiss/copy buttons: `text-red-400/70 hover:text-red-300`
- Success toast icon/text: `text-emerald-300`
- These are the only locations where palette classes appear in Phase 5 components.

## Component Inventory

### New Components (added to `src/components/ui.rs`)

#### ToastEntry (data struct)
```
ToastEntry { id: u64, kind: ToastKind, message: String }
ToastKind: Success | Error
```

#### ToastContext (context struct)
```
ToastContext { show: Callback<(ToastKind, String)> }
```
— `Callback` is `Copy`; pages extract via `use_context::<ToastContext>()`.

#### ToastProvider (wraps App in `app.rs`)
- Fixed overlay at `top-4 left-1/2 -translate-x-1/2 z-50`
- Flex column, gap-2, items-center, pointer-events-none container
- Success toasts auto-dismiss after 4000ms via `set_timeout`
- Error toasts persist until dismissed
- Stack behavior: vertical column, newest on top, max 3 visible (oldest success removed first)

#### Toast item visual spec
| State | Background | Border | Text color | Min width | Border radius |
|-------|-----------|--------|-----------|-----------|---------------|
| Success | `bg-emerald-500/20` | `border-emerald-500/40` | `text-emerald-300` | `min-w-64` | `rounded-xl` |
| Error | `bg-red-500/20` | `border-red-500/40` | `text-red-300` | `min-w-64` | `rounded-xl` |

Error toast extra controls: "Copy" button + "×" dismiss button — `text-xs px-1.5 py-0.5 rounded hover:bg-red-500/20 transition-colors`

Toast animation: `transition-opacity` fade-in (CSS default). No slide animation — avoids complexity with Leptos reactive list. Opacity is implicit from DOM insertion.

#### SkeletonLine
- Props: `width: &'static str` (Tailwind w-* class, default `w-full`), `height: &'static str` (default `h-4`)
- Classes: `animate-pulse bg-elevated rounded {width} {height}`

#### SkeletonCard
- Props: `height: &'static str` (default `h-24`)
- Classes: `animate-pulse bg-elevated rounded-xl border border-divider/30 {height} w-full`

#### SkeletonGrid
- Props: `cols: u8` (2/3/4, default 3), `rows: u8` (default 2), `card_height: &'static str` (default `h-20`)
- Grid class map: 2→`grid-cols-2`, 3→`grid-cols-3`, 4→`grid-cols-4`
- Gap: `gap-3` between all cells

#### EmptyState (enhanced — backward-compatible)
- Props: `message: &'static str` (required), `icon: Option<&'static str>`, `cta_label: Option<&'static str>`, `cta_href: Option<&'static str>`
- Layout: `text-center py-12 flex flex-col items-center gap-3`
- Icon: `text-4xl` (Unicode character rendered as text)
- Message: `text-secondary text-sm max-w-xs`
- CTA: `bg-accent hover:bg-accent-hover text-accent-contrast font-semibold rounded-lg px-4 py-2 text-sm transition-colors`
- No CTA rendered when either `cta_label` or `cta_href` is None

#### NoTeamState (zero-prop convenience wrapper)
- Renders EmptyState with icon="👥", message fixed (see Copywriting), cta to `/team/roster`

### Error States (load failures — ErrorBanner, not toast)

ErrorBanner is KEPT for `Err` branches inside `<Suspense>` content — these are structural load failures, not mutation feedback.

| Condition | Copy |
|-----------|------|
| Generic load failure | "Failed to load — {server error message}" |

ErrorBanner is NOT migrated to toast. It remains in `src/components/ui.rs` unchanged.

### Destructive Actions

No new destructive actions are introduced in Phase 5. Existing delete actions (drafts, reviews, game plans, action items) already have their own confirmation patterns. Phase 5 adds success toasts to these existing flows — no new confirmation dialogs required.

## Interaction Contracts

### Toast System

| Interaction | Behavior |
|-------------|----------|
| Success toast appears | Fades in at top-center, fixed overlay, z-50 |
| Success toast auto-dismiss | After 4000ms via `set_timeout` + `Closure::once` |
| Error toast appears | Same position; persists until manual dismiss |
| Error "Copy" clicked | `navigator().clipboard().write_text(msg)` — promise discarded |
| Error "×" clicked | `set_toasts.update(retain)` removes that toast by id |
| Multiple toasts | Stack vertically (flex col gap-2), newest at top, max 3 visible |
| 4th toast arrives (success) | Oldest success toast removed immediately before adding new |
| 4th toast arrives (error) | All 4 shown — errors never auto-removed by stacking logic |

### Empty States

| Interaction | Behavior |
|-------------|----------|
| CTA button clicked | Standard `<a href=...>` navigation (no JS needed) |
| Anchor CTA clicked | Browser scroll to element ID |

### Post-Game Action Item Banner

| Interaction | Behavior |
|-------------|----------|
| Review saved with improvements | Banner appears above editor with count and "View" link |
| "View" link clicked | Navigate to `/action-items` |
| Next save triggered | Previous banner replaced with new count (or hidden if 0 new) |
| Banner dismiss | No explicit dismiss button — replaced on next save |

### Skeleton Loading

| Interaction | Behavior |
|-------------|----------|
| Suspense pending | Skeleton shown immediately (no delay) |
| Data resolves | Skeleton replaced by content (Leptos Suspense default) |
| No interaction on skeleton | Skeletons are non-interactive (no click/hover states) |

<objective>
Add pool warning badges to the draft planner. When a champion is picked in a slot and the assigned player does not have that champion in their pool (and it exposes a class gap), a warning badge appears at the top-right of the pick slot. Warnings are computed client-side using already-loaded team pool data and champion metadata — no server calls on pick changes.

Purpose: Coaches see at a glance when a draft pick assigns a champion outside a player's comfort zone, enabling real-time draft adjustments (PIPE-03).
Output: Modified `draft_board.rs` with optional `warning_slots` prop, modified `draft.rs` with `compute_slot_warnings` Memo and role-override dropdown.
</objective>

   - Returns: `Vec<Option<String>>` of length 20
   - Logic:
     a. Determine our pick slot indices based on side: blue picks = [6, 9, 10, 17, 18], red picks = [7, 8, 11, 16, 19]

     d. Find the team member whose role matches (from pools tuple)
     e. If slot has a champion and that champion ID is NOT in the matched player's pool entries, set `Some(username.clone())`
     f. For non-our-side slots and ban slots, return `None`

# Phase 18 — UI Design Contract: Region Variants

> Visual and interaction contract for Phase 18. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note (from CLAUDE.md):** This spec covers Phase 18-specific structural decisions only —
> primitive component API, mode toggle interactions, visual-regression contracts, page-pair
> inventory, and region-aware copywriting. Tokens, color palette, typography scale, and
> accessibility rules are already defined in the vault wiki and `input.css`. They are
> referenced here but NOT redefined.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn; standalone Tailwind v4 binary) |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG (`region/chrome.rs::Icon`); CommunityDragon SVGs for role icons |
| Fonts | Cinzel · Cormorant Garamond · VT323 · Inter · JetBrains Mono (all self-hosted; no new fonts) |

**Design system source:** CSS custom properties via `@theme` in `input.css`. Two region tokens
sets at `input.css:193-265` (`[data-theme="demacia"]` / `[data-theme="pandemonium"]`). Both
already exist from Phase 17. Phase 18 does not add or change token definitions.

**Primitive architecture:** ~24 region-aware Leptos components under `src/components/region/`
(7 submodules: `typography`, `ornaments`, `layout`, `controls`, `data_viz`, `solo`, `chrome`).
All consume semantic tokens (`bg-accent`, `text-primary`, etc.) via CSS custom properties.
Raw hex is forbidden in `src/components/` and `src/pages/` per project guardrails.

## Spacing Scale

Inherited from vault design system (8-point scale). Not redefined in this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing (tile gaps, stat card internals) |
| md | 16px | Default element spacing, card padding base |
| lg | 24px | Card padding, section internal spacing |
| xl | 32px | Between cards, column gaps |
| 2xl | 48px | Section breaks, major panel margins |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Champion tiles: 56px × 56px (non-multiple of 8 for visual density; sourced from `ChampTile` in `_shared/components.jsx`)
- Role icon touch target: 44px minimum (accessibility, G-12 compliant)
- GiltCorner ornament: 20px (for `variant="gilt"` card corners); 28px standalone

## Primitive Component API Contract

Every primitive in `src/components/region/` follows this signature pattern:

```
fn PrimitiveName(
    region: String,          // "demacia" | "pandemonium" — passed from page entry, never use_context inside
    #[prop(optional)] variant: Option<String>,  // e.g. "gilt" | "zine" | "default"
    #[prop(optional)] tone: Option<String>,     // e.g. "accent" | "neutral" | "success" | "danger"
    // ... primitive-specific props
) -> impl IntoView
```

**Branching pattern** (sourced from `src/components/ornaments.rs:128-167`):

```rust
let is_pandemonium = region == "pandemonium";
move || if is_pandemonium {
    view! { /* pandemonium variant */ }.into_any()
} else {
    view! { /* demacia variant (default) */ }.into_any()
}
```

No `use_context::<InitialTheme>()` inside primitives. Pages read context ONCE at entry and pass
`region: String` down as a prop.

### Submodule inventory

**`region/typography.rs`** — 6 type primitives:
- `Display { region, size: "xl"|"lg"|"md"|"sm", children }`
- `Imperial { region, children }`
- `H { region, level: 1|2|3, children }`
- `Eyebrow { region, children }`
- `Mono { children }` — region-neutral (same font both regions)
- `Glitch { children }` — Pandemonium-only VT323; Demacia fallback: `Eyebrow` treatment

**`region/ornaments.rs`** — all ornaments consolidated here (replaces deleted `src/components/ornaments.rs`):
- `GiltCorner { size: u32, corner: "tl"|"tr"|"bl"|"br" }` — Demacia only
- `HeraldicDivider { region, width: u32 }` — renders divider for Demacia, `RiotTape` for Pandemonium
- `RiotTape { width: u32, label: String }` — Pandemonium only
- `FleurDeLis { size: u32 }` — moved from old ornaments.rs, Demacia only
- `Crown { size: u32 }` — moved from old ornaments.rs
- `CompanionSigil { region, size: u32 }` — moved from old ornaments.rs, region-branching already exists

**`region/layout.rs`** — 3 layout primitives:
- `Card { region, variant: "default"|"gilt"|"zine", children }` — gilt = Demacia inset border + GiltCorners; zine = Pandemonium bracket corners + borderRadius 0
- `SectionHead { region, eyebrow: Option<String>, title: String, action: Option<impl IntoView> }` — Demacia: Eyebrow + H level 2; Pandemonium: Glitch `// eyebrow` + H bold italic uppercase
- `Themed { region, children }` — sets `[data-theme]` attribute on wrapper div

**`region/controls.rs`** — 2 interactive primitives:
- `Btn { region, variant: "primary"|"secondary"|"outline"|"ghost", size: "sm"|"md"|"lg", children }` — Demacia primary: gilt gradient + Cinzel uppercase; Pandemonium primary: magenta + mono uppercase + offset shadow
- `Badge { tone: "neutral"|"accent"|"success"|"warning"|"danger"|"info", children }` — region-neutral (same appearance both regions); uses JetBrains Mono

**`region/data_viz.rs`** — 3 data primitives:
- `Stat { label: String, value: String, unit: Option<String>, delta: Option<f64> }` — region-neutral display; Eyebrow label + Mono value
- `Sparkline { data: Vec<f64>, width: u32, height: u32 }` — region-neutral SVG line + gradient fill
- `MoodMeter { value: f64 }` — 5-segment bar; 0.0–1.0 maps to filled/empty segments; color via success/accent/warning tokens

**`region/solo.rs`** — 2 solo-specific primitives:
- `RankBadge { tier: String, division: String, large: bool }` — SVG shield crest with tier color
- `LPProgress { region, lp: u32, max: u32 }` — Demacia: gold gradient fill, rounded bar; Pandemonium: flat magenta fill, square bar

**`region/chrome.rs`** — 4 champion/role chrome primitives:
- `ChampPortrait { name: String, size: u32, kind: "square"|"loading"|"splash" }` — DDragon image with error fallback to elevated bg
- `ChampTile { name: Option<String>, size: u32, banned: bool, locked: bool, dimmed: bool }` — 56×56 default; banned: grayscale + diagonal bar; locked: accent dot
- `RoleIcon { role: String, size: u32 }` — CommunityDragon SVG via CSS mask; role ∈ {top, jungle, mid, adc, support}
- `Icon { name: String, size: u32 }` — inline SVG icon set (home, user, swords, tree, chart, etc.)

**`region/mod.rs`** — `pub use` re-exports for all 7 submodules.

## Mode Toggle Interaction Contract

### Persistence model

Three new DB fields on `user` table (added in 18-08):

```surql
DEFINE FIELD IF NOT EXISTS draft_mode ON user TYPE string DEFAULT 'auto';
DEFINE FIELD IF NOT EXISTS team_dashboard_mode ON user TYPE string DEFAULT 'auto';
DEFINE FIELD IF NOT EXISTS solo_mode ON user TYPE string DEFAULT 'auto';
```

`'auto'` sentinel = region-coupled default (resolved at page entry). Once the user clicks a
toggle, the explicit value is stored and persists across region switches.

**Resolver** (implemented as a Rust function, not a macro):

```
resolve_mode(stored: &str, region: &str, route) -> &str
  if stored != "auto" { return stored }
  match (route, region) {
    (Draft,         "demacia")     => "carousel"
    (Draft,         "pandemonium") => "ledger"
    (TeamDashboard, "demacia")     => "dashboard"
    (TeamDashboard, "pandemonium") => "brief"
    (Solo,          "demacia")     => "constellation"
    (Solo,          "pandemonium") => "forge"
    _                              => "carousel"
  }
```

Note: `solo` Pandemonium default is `forge` (tactical/workbench). `journal` is the reflection
mode. Confirm can be revisited during 18-08 once `solo-forge/pandemonium.jsx` is reviewed —
but `forge` is the locked default per CONTEXT.md D-04.

### Toggle component shape

The mode toggle is an in-page control, NOT a URL parameter. It renders inside the page body
(not in the nav). The exact widget shape (segmented control / stamped tab strip) is left to
gsd-planner for 18-08, with this constraint: the toggle MAY itself be region-aware (Demacia
segmented control with gilt borders; Pandemonium rubber-stamp tab pull) — planner decides.

## Region-Diff Spec Contract (`e2e/tests/region-diff.spec.ts`)

New spec created in 18-09. Its single responsibility: assert that each scoped route renders
visually distinct output between Demacia and Pandemonium.

### Assertion contract

For every scoped route + mode combination:

```typescript
const demaciaBuffer = await captureRegion(page, route, mode, 'demacia');
const pandemoniumBuffer = await captureRegion(page, route, mode, 'pandemonium');
const ratio = pixelDiffRatio(demaciaBuffer, pandemoniumBuffer);
expect(ratio).toBeGreaterThan(0.40);
```

`pixelDiffRatio` uses `pixelmatch` with `threshold: 0.1` (same library Playwright uses
internally). The 0.40 threshold means at least 40% of pixels must differ between regions on
every scoped page.

### Routes covered by region-diff spec

| Route | Modes tested |
|-------|-------------|
| `/draft` | carousel, war-table, ledger |
| `/solo` | constellation, forge, journal |
| `/team/dashboard` | dashboard, brief |
| `/tree-drafter` | (no mode) |
| `/champion-pool` | (no mode) |
| `/match/:id` | (no mode, synthetic ID) |
| `/post-game` | (no mode) |

PageLoading skeletons: at minimum `draft` loading variant tested for both regions in the
region-diff spec (supports SPEC Requirement 2 acceptance criterion).

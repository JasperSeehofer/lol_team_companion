Add the following to src/components/ui.rs, keeping existing ErrorBanner, StatusMessage, and EmptyState (enhanced):

**1. Toast data types:**
```rust
#[derive(Clone, Debug, PartialEq)]
pub enum ToastKind { Success, Error }

#[derive(Clone, Debug)]
pub struct ToastEntry { pub id: u64, pub kind: ToastKind, pub message: String }
```

**2. ToastContext struct:**
```rust
#[derive(Clone, Copy)]
pub struct ToastContext { pub show: Callback<(ToastKind, String)> }
```

**3. ToastProvider component** — owns `signal(Vec::<ToastEntry>::new())`, a `StoredValue::new(std::cell::Cell::new(0u64))` for ID generation. Creates a `Callback::new(move |(kind, message)| { ... })` that:
- Generates incrementing ID
- Pushes ToastEntry to the signal
- If kind == Success, schedules `set_timeout_with_callback_and_timeout_and_arguments_0` at 4000ms to remove that toast by ID (guarded by `#[cfg(feature = "hydrate")]`)
- Enforces max 3 visible: before pushing, if toasts.len() >= 3, remove oldest Success toast (if any)
- Calls `provide_context(ToastContext { show })`
- Renders `{children()}` followed by `<ToastOverlay toasts set_toasts />`

**4. ToastOverlay component** — fixed overlay:
- Container: `"fixed top-4 left-1/2 -translate-x-1/2 z-50 flex flex-col gap-2 items-center pointer-events-none"`
- Maps over `toasts.get()`, rendering each toast with:
  - Success style: `"pointer-events-auto flex items-center gap-3 bg-emerald-500/20 border border-emerald-500/40 text-emerald-300 rounded-xl px-4 py-3 text-sm shadow-lg min-w-64 max-w-sm"`
  - Error style: same but `bg-red-500/20 border border-red-500/40 text-red-300` and uses `items-start` instead of `items-center`
  - Error toasts get "Copy" button (`navigator().clipboard().write_text(&msg)` guarded by `#[cfg(feature = "hydrate")]`) and "x" dismiss button
  - Button styles: `"text-red-400/70 hover:text-red-300 text-xs px-1.5 py-0.5 rounded hover:bg-red-500/20 transition-colors"`
  - Use `.into_any()` on each if/else branch for error vs success controls
  - Use `.collect_view()` on the iterator

**5. Skeleton primitives** — add three components:
- `SkeletonLine` with props `width: &'static str` (default `"w-full"`), `height: &'static str` (default `"h-4"`). Classes: `format!("animate-pulse bg-elevated rounded {width} {height}")`
- `SkeletonCard` with prop `height: &'static str` (default `"h-24"`). Classes: `format!("animate-pulse bg-elevated rounded-xl border border-divider/30 {height} w-full")`
- `SkeletonGrid` with props `cols: u8` (default 3), `rows: u8` (default 2), `card_height: &'static str` (default `"h-20"`). Maps cols to grid class (`2=>"grid grid-cols-2 gap-3"`, `3=>"grid grid-cols-3 gap-3"`, `4=>"grid grid-cols-4 gap-3"`, else `"grid grid-cols-1 gap-3"`). Creates `Vec<u8>` of `cols*rows` items, maps to `<SkeletonCard height=card_height />`.

**6. Enhanced EmptyState** — replace existing EmptyState, making icon/CTA optional:
```rust
#[component]
pub fn EmptyState(
    message: &'static str,
    #[prop(optional)] icon: Option<&'static str>,
    #[prop(optional)] cta_label: Option<&'static str>,
    #[prop(optional)] cta_href: Option<&'static str>,

```
- Layout: `"text-center py-12 flex flex-col items-center gap-3"`
- If icon is Some: render `<span class="text-4xl">{icon}</span>`
- Message: `<p class="text-secondary text-sm max-w-xs">{message}</p>`
- If both cta_label and cta_href are Some: render `<a href=href class="mt-1 bg-accent hover:bg-accent-hover text-accent-contrast font-semibold rounded-lg px-4 py-2 text-sm transition-colors">{label}</a>`
- Use `.into_any()` on branches

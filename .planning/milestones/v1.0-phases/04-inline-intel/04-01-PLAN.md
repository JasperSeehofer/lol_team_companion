<objective>
Add pool warning badges to the draft planner. When a champion is picked in a slot and the assigned player does not have that champion in their pool AND the champion's class tags expose a class gap for that player, a warning badge appears at the top-right of the pick slot. Warnings are computed client-side using already-loaded team pool data and champion metadata -- no server calls on pick changes.

Purpose: Coaches see at a glance when a draft pick assigns a champion outside a player's comfort zone with coaching-quality insight about class coverage gaps, enabling real-time draft adjustments (PIPE-03).
Output: Modified `draft_board.rs` with optional `warning_slots` prop, modified `draft.rs` with `compute_slot_warnings` Memo (with class gap logic and unit tests) and role-override dropdown.

1. **Define `compute_slot_warnings` function** (pure, no async, `#[cfg(not(target_arch = "wasm32"))]` is NOT needed -- this runs on both targets) at the top of draft.rs (or in a helper block). This function:
   - Takes: `slots: &[Option<String>]` (20 draft slots), `pools: &[(String, String, Vec<ChampionPoolEntry>)]` (username, role, pool entries), `champions: &HashMap<String, Champion>` (champion data for class tags), `our_side: &str` ("blue" or "red"), `role_overrides: &HashMap<usize, String>` (manual slot-role assignments)
   - Returns: `Vec<Option<(String, String)>>` of length 20 -- `Some((player_name, class_gap_detail))`
   - Logic:
     a. Determine our pick slot indices based on side: blue picks = [6, 9, 10, 17, 18], red picks = [7, 8, 11, 16, 19]
     b. Map pick order to default roles: [top, jungle, mid, bot, support] (conventional draft order)
     c. For each our-side pick slot: check role_overrides first, then use default role
     d. Find the team member whose role matches (from pools tuple)
     e. If slot has a champion:
        - Check if champion ID is in the matched player's pool entries. If YES -> no warning (None).
        - If NOT in pool: check the champion's class tags (from `champions` map, using `Champion.tags`). Collect the set of classes covered by the player's existing pool (iterate their pool entries, look up each entry's champion in `champions` map, collect all tags). Determine which of the picked champion's class tags are NOT covered by the player's pool classes. If any class tag is missing from the player's pool -> emit `Some((username, "No {MissingClass} coverage"))` using the first missing class. If ALL of the picked champion's class tags are already covered by the player's pool -> no warning (None), because there's no class gap even though the specific champion isn't in the pool.
     f. For non-our-side slots and ban slots, return `None`
   - Per user decision in CONTEXT.md: "Pool warnings should combine 'not in pool' with class gap intel -- not just a binary yes/no, but coaching-quality insight."
   - Do NOT make server calls. Do NOT use `Resource`. This is pure computation.

   #[cfg(test)]
   mod tests {
       use super::*;
       use crate::models::champion::{Champion, ChampionPoolEntry};
       use std::collections::HashMap;

       fn make_champion(id: &str, tags: Vec<&str>) -> Champion {
           Champion {
               id: id.to_string(),
               name: id.to_string(),
               title: String::new(),
               tags: tags.into_iter().map(String::from).collect(),
               image_full: String::new(),
           }
       }

       fn make_pool_entry(user_id: &str, champion: &str, role: &str) -> ChampionPoolEntry {
           ChampionPoolEntry {
               user_id: user_id.to_string(),
               champion: champion.to_string(),
               role: role.to_string(),
               tier: "S".to_string(),
               notes: None,
               comfort_level: None,
               meta_tag: None,
           }
       }

       #[test]
       fn champion_in_pool_no_warning() {
           // Player has Darius in pool -> picking Darius = no warning
           let mut champs = HashMap::new();
           champs.insert("Darius".to_string(), make_champion("Darius", vec!["Fighter"]));
           let pools = vec![("TopPlayer".to_string(), "top".to_string(),
               vec![make_pool_entry("user:u1", "Darius", "top")])];
           let mut slots: Vec<Option<String>> = vec![None; 20];
           slots[6] = Some("Darius".to_string()); // Blue pick 1 = top
           let result = compute_slot_warnings(&slots, &pools, &champs, "blue", &HashMap::new());
           assert!(result[6].is_none(), "Champion in pool should not trigger warning");
       }

       #[test]
       fn champion_not_in_pool_with_class_gap_warns() {
           // Player has only Fighters, picks a Mage (not in pool + class gap)
           let mut champs = HashMap::new();
           champs.insert("Darius".to_string(), make_champion("Darius", vec!["Fighter"]));
           champs.insert("Syndra".to_string(), make_champion("Syndra", vec!["Mage"]));
           let pools = vec![("TopPlayer".to_string(), "top".to_string(),
               vec![make_pool_entry("user:u1", "Darius", "top")])];
           let mut slots: Vec<Option<String>> = vec![None; 20];
           slots[6] = Some("Syndra".to_string()); // Blue pick 1 = top
           let result = compute_slot_warnings(&slots, &pools, &champs, "blue", &HashMap::new());
           assert!(result[6].is_some(), "Not-in-pool champion with class gap should warn");
           let (name, detail) = result[6].as_ref().unwrap();
           assert_eq!(name, "TopPlayer");
           assert!(detail.contains("Mage"), "Should mention the missing class");
       }

       #[test]
       fn champion_not_in_pool_without_class_gap_no_warning() {
           // Player has Darius (Fighter) in pool, picks Garen (also Fighter, not in pool)
           // No class gap because Fighter is already covered
           let mut champs = HashMap::new();
           champs.insert("Darius".to_string(), make_champion("Darius", vec!["Fighter"]));
           champs.insert("Garen".to_string(), make_champion("Garen", vec!["Fighter"]));
           let pools = vec![("TopPlayer".to_string(), "top".to_string(),
               vec![make_pool_entry("user:u1", "Darius", "top")])];
           let mut slots: Vec<Option<String>> = vec![None; 20];
           slots[6] = Some("Garen".to_string());
           let result = compute_slot_warnings(&slots, &pools, &champs, "blue", &HashMap::new());
           assert!(result[6].is_none(), "No class gap = no warning even if champion not in pool");
       }

       #[test]
       fn wrong_side_slot_no_warning() {
           // Blue side: red pick slots should never have warnings
           let mut champs = HashMap::new();
           champs.insert("Syndra".to_string(), make_champion("Syndra", vec!["Mage"]));
           let pools = vec![("TopPlayer".to_string(), "top".to_string(), vec![])];
           let mut slots: Vec<Option<String>> = vec![None; 20];
           slots[7] = Some("Syndra".to_string()); // Red pick 1 (not our side when blue)
           let result = compute_slot_warnings(&slots, &pools, &champs, "blue", &HashMap::new());
           assert!(result[7].is_none(), "Opponent-side slots should not have warnings");
       }

       #[test]
       fn side_switch_changes_checked_slots() {
           let mut champs = HashMap::new();
           champs.insert("Syndra".to_string(), make_champion("Syndra", vec!["Mage"]));
           let pools = vec![("TopPlayer".to_string(), "top".to_string(),
               vec![make_pool_entry("user:u1", "Darius", "top")])];
           // Darius not in champs map, but in pool. Syndra not in pool.
           champs.insert("Darius".to_string(), make_champion("Darius", vec!["Fighter"]));
           let mut slots: Vec<Option<String>> = vec![None; 20];
           slots[6] = Some("Syndra".to_string()); // Blue pick 1
           slots[7] = Some("Syndra".to_string()); // Red pick 1
           let blue_result = compute_slot_warnings(&slots, &pools, &champs, "blue", &HashMap::new());
           let red_result = compute_slot_warnings(&slots, &pools, &champs, "red", &HashMap::new());
           // Blue side: slot 6 is ours (may warn), slot 7 is theirs (no warn)
           // Red side: slot 7 is ours (may warn), slot 6 is theirs (no warn)
           assert!(blue_result[7].is_none(), "Slot 7 is opponent on blue side");
           assert!(red_result[6].is_none(), "Slot 6 is opponent on red side");
       }

           assert_eq!(result.len(), 20);
           assert!(result.iter().all(|w| w.is_none()));
       }
   }

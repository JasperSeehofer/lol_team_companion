## Color

The project uses semantic tokens defined in `input.css`. Phase 6 makes no changes to color. All bug fixes must use only existing semantic tokens — no hardcoded hex or new color values.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-base` | Page background (`oklch(0.13 0.004 264.665)` dark default) |
| Secondary (30%) | `bg-surface` / `bg-elevated` | Cards, nav, sidebar, panels |
| Accent (10%) | `bg-accent` / `text-accent` | Yellow-400 default; user-selectable palette |
| Destructive | `bg-red-500/10` + `text-red-400` | Error banners, error toasts only |

## Interaction Contract

These are the corrected interaction states that Phase 6 must deliver. Each maps to a specific bug fix.

### BUG-01: Tree Drafter Branch Freeze

**Before fix:** Clicking "Branch from here" freezes the editor — labels cannot be typed, save button does not respond.

**After fix:** After branching, the new node is selected and the label input is immediately editable. The auto-save timer starts fresh with no pending stale saves. The UI remains fully responsive throughout.

**Verification signal:** User types a character in the label field within 500ms of branch completing and the character appears in the input.

### BUG-02 / PLAN-02: "View Game Plan" Navigation

**Before fix:** "View Game Plan" link navigates to `/game-plan` with no query params, showing an empty or wrong game plan.

**After fix:** The link navigates to `/game-plan?plan_id={existing_plan_id}`, loading the specific game plan linked to the draft with its data pre-populated.

**URL pattern:** `/game-plan?plan_id=<id>` — exact query param key to be confirmed against `game_plan.rs` before implementing.

### BUG-03: Team Leader Roster Visibility

**Before fix:** Team leader disappears from roster list under certain `role`/`roster_type` combinations.

**After fix:** The team leader is always visible in the roster list regardless of their `role` or `roster_type` value. The leader slot renders at the top of the member list or is always included in whichever partition renders first.

### BUG-04: Hydration Mismatch Warnings

**Before fix:** Browser console shows hydration mismatch warnings on page load because `nav.rs` reads the `user` Resource outside a Suspense boundary.

**After fix:** No hydration mismatch warnings appear in the browser console on any page. The nav renders correctly in both SSR and WASM hydration passes.

**Visual parity requirement:** The nav must render identically (same links visible/hidden) in SSR output and after hydration. No flash of wrong auth state.

### BUG-05: Draft Drag-and-Drop Move Behavior

**Before fix:** Dragging a filled draft slot copies the champion icon (browser drags the `<img>` src) rather than moving it. The source slot retains its champion after the drag.

**After fix:** Dragging a filled slot moves the champion to the target slot and clears the source slot. Only one slot holds the champion at a time.

**Visual states required for BUG-05:**
- **Draggable filled slot:** Add `draggable="true"` — no visual change at rest.
- **Drag-in-progress on source:** Browser default drag ghost is acceptable; no custom drag image required.
- **Valid drop target (drag over):** `border-accent/50` highlight on the target slot while dragging over it.
- **After drop:** Target slot shows the champion; source slot returns to empty state (same visual as an unfilled slot).

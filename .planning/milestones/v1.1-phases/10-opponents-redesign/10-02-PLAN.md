    ```rust
    pub struct PlayerIntelData {
        pub recent_champions: Vec<String>,             // unique champion names (same as fetch_player_champions output)
        pub champion_with_role: Vec<(String, String)>, // (champion_name, team_position) per match — NOT deduplicated
        pub mastery_data: Vec<(String, i32, i32)>,     // (champion_name, mastery_level, mastery_points) from fetch_champion_masteries
    }
    ```

    **Add function:**
    ```rust
    pub async fn fetch_player_intel(puuid: &str, match_count: usize) -> Result<PlayerIntelData, RiotError> {
    ```
    Implementation:
    1. Fetch match IDs: same as `fetch_player_champions` — `api.match_v5().get_match_ids_by_puuid(RegionalRoute::EUROPE, puuid, Some(match_count as i32), None, None, None, None, None).await?`
    2. Loop over match IDs, for each:
       - `api.match_v5().get_match(RegionalRoute::EUROPE, &mid).await?`
       - Find participant: `m.info.participants.iter().find(|p| p.puuid == puuid)`
       - Extract `champion_name` (String) and `team_position` (String — Riot API values: "TOP", "JUNGLE", "MIDDLE", "BOTTOM", "UTILITY")
       - Push `(champion_name.clone(), team_position)` to `champion_with_role` vec (every match, not deduplicated)
       - Track unique champions in a `HashSet<String>` for `recent_champions` vec (same dedup as existing fn)
    3. After match loop, call `fetch_champion_masteries(puuid).await?` for mastery data
    4. Return `PlayerIntelData { recent_champions, champion_with_role, mastery_data }`

    Note on `team_position` access: The riven `Participant` struct has `pub team_position: String` field. Access it directly as `p.team_position.clone()`. If the field is empty string for some game modes, that's fine — the role distribution computation handles empty strings by skipping them.

    Use `std::collections::HashSet` for dedup (already the pattern in `fetch_player_champions`).
